# NASネットワーク接続問題解決履歴

**実施日**: 2025年9月12日  
**問題**: NAS設定後のWiFi接続不安定化

## 問題の概要

### 症状
- NAS設定後にパソコンのWiFi接続が不安定になる
- 接続の切断や速度低下が発生

### 原因分析
NAS設定時に使用したUSB有線LANアダプタが引き続き有効な状態で、WiFiとの間でネットワーク優先順位の競合が発生していたことが判明。

## ネットワーク状況調査

### 発見された問題
1. **複数のアクティブ接続**
   - WiFi (en0): 10.158.1.168
   - USB LAN (en6): 169.254.1.1 (Link-Local)
   
2. **ネットワーク優先順位の競合**
   - 有線と無線が同時にアクティブ
   - システムが最適な経路を選択できない状態

### 接続状況
```
WiFi接続情報:
- SSID: DK-home _main
- 信号強度: -38 dBm (良好)
- 速度: 229 Mbps (802.11ax)
- チャンネル: 1 (2GHz, 20MHz)
- セキュリティ: WPA2/WPA3 Personal

有線LAN接続:
- インターフェース: en6 (USB 10/100/1000 LAN)
- IP: 169.254.1.1 (Link-Local)
- 状態: アクティブ
```

## 実施した解決策

### 1. ネットワーク優先順位の設定
WiFiを最優先に設定し、有線接続を下位に配置：
```bash
networksetup -ordernetworkservices "Wi-Fi" "USB 10/100/1000 LAN" "USB 10/100/1000 LAN 3" "USB 10/100/1000 LAN 2" "USB 10/100/1000 LAN 4" "Thunderbolt Bridge" "ATEM Mini Extreme ISO"
```

### 2. DNSキャッシュクリア
ネットワーク設定変更を確実に反映させるためキャッシュをクリア：
```bash
dscacheutil -flushcache
```

### 3. 接続安定性テスト
解決後の動作確認：
```bash
ping -c 10 8.8.8.8
```

### 最終的なネットワーク構成
```
優先順位:
1. Wi-Fi (メイン接続)
2. USB 10/100/1000 LAN (NAS用、必要時のみ)
3. その他のネットワークサービス
```

## 採用した運用方針

### 設定方針の決定
以下の2つの選択肢から**方針1**を採用：

**方針1: 優先順位変更のみ（採用）**
- ✅ WiFiを最優先に設定
- ✅ USB LANは有効だが低優先度
- ✅ 必要時にNAS直接接続が可能
- ✅ ユーザーの柔軟性を保持

**方針2: 有線接続完全無効化（不採用）**
- USB LANを完全に無効化
- NAS直接接続時に再設定が必要

### トラブルシューティング手順
問題が再発した場合の対処法：

1. **優先順位確認**
   ```bash
   networksetup -listallnetworkservices
   ```

2. **接続状況確認**
   ```bash
   ifconfig
   netstat -rn
   ```

3. **有線接続無効化（最終手段）**
   ```bash
   sudo networksetup -setnetworkserviceenabled "USB 10/100/1000 LAN" off
   ```
