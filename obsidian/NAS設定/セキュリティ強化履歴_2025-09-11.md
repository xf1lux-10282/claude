# NASセキュリティ強化履歴

**実施日**: 2025年9月11日  
**対象**: NAS (10.158.1.88 / 169.254.1.2)

## 実施したセキュリティ強化

### 1. 初期セキュリティ診断
**発見された問題:**
- SMBポート(139, 445)が開放
- Web管理画面(9999, 9443)への制限なし
- fail2banが未設定状態

**開放ポート一覧:**
- ポート80: HTTP → 9999リダイレクト
- ポート139: NetBIOS
- ポート443: HTTPS → 9443リダイレクト  
- ポート445: SMB/CIFS
- ポート9999: Web管理画面
- ポート22000: Syncthing

### 2. SMB/CIFS設定強化
```
✅ SMBプロトコル: SMBv2, SMBv2.1, SMBv3のみ (SMBv1無効化)
✅ 伝送暗号化: 強制
✅ サーバー署名: 強制
✅ WINSサーバー: 無効化
```

### 3. ファイアウォール設定
**新規ルール作成:**
```
ルール1: SMB_Local_Only
- ポート: 139, 445
- プロトコル: TCP
- アクション: 許可
- 送信元IP: 10.158.1.0/24

ルール2: Admin_Panel_Restrict
- ポート: 9999, 9443
- プロトコル: TCP
- アクション: 許可
- 送信元IP: 10.158.1.168
```

### 4. 認証強化
- ✅ 2段階認証: 既に有効化済み
- ✅ パスワード複雑度: 確認済み

### 5. fail2ban侵入防止システム
**Docker環境での設定:**
```yaml
環境変数:
- PUID: 1000
- PGID: 10
- TZ: Asia/Tokyo
- ネットワーク: host

jail設定 (/data/jail.d/nas-protection.local):
[DEFAULT]
bantime = 3600
findtime = 600
maxretry = 3
ignoreip = 127.0.0.1/8 10.158.1.168

[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
```

**動作確認:**
- jail数: 1
- 保護対象: SSH (sshd)
- 外部SSH接続: ブロック確認済み
