**決定事項（と理由）:**
- **実行基盤は GitHub Actions（自前サーバー不要）** … PCを起動し続ける必要がなく、無料枠（このジョブは月約360分）に収まり、管理コストが最小だから。VPSは「数分おきの高頻度」「ヘッドレスブラウザ常駐が必要」等の要件が出た時に再検討。
- **通知は ntfy / Discord 両対応** … どちらもスマホ/PCの公式アプリにプッシュが届き、ユーザーの「アプリで受け取りたい」要望に最も手軽に応えられるため。LINE Notifyは2025年終了のため不採用。
- **ダッシュボードは PWA（GitHub Pages）** … ネイティブアプリ自作は大がかり。PWAならホーム画面に追加して“アプリのように”使え、データはActionsが自動更新するため。
- **グラフは階段状（stepped）** … 価格は「普段一定→社会情勢で稀に段差状に変化」する性質。いつ変わったかを一目で把握するため滑らかな線から変更。
- **通知は商品価格が変わった時だけ。為替は記録のみ** … 為替は毎日動くので通知すると煩い。「USD価格の変化＝注文タイミングのシグナル」を保ちつつ、円コストは後から振り返れる形に。
- **チェック頻度は日本時間6/9/12/15/18/21時** … ユーザー指定。cronはUTC換算（21,0,3,6,9,12時）。
- **目標価格(target_price)は不使用** … ユーザーの目的は「特定額への到達」ではなく「価格の上下の察知」と「推移の振り返り」だったため全商品null。
