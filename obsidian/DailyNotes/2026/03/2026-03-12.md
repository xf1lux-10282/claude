## ColorBlocker v3.0 解析セッション

### 実施内容

1. **EDN ColorBlockerの仕組みを完全解析**
   - Webサイト http://www.easydigitalnegatives.com/color-blocker/ の内容確認
   - デスクトップの `EDN_ColorBlocker_3` フォルダー内のプログラム解析
   - JavaScriptコード ([funk.js](file:///Users/daisukekinoshita/Desktop/EDN_ColorBlocker_3/fold/funk.js)) の動作原理を理解

2. **Maximum Blocker vs Optimum Blocker**
   - **Maximum Blocker**: 最もUV光を強く遮断する色（UV遮断力最大）
   - **Optimum Blocker**: 最も均等にUV光を遮断する色（階調性とUV遮断のバランス最良）
   - プラチナプリントでは Maximum Blocker が推奨

3. **測定プロセスの理解**
   ```
   1. EDN_HSB2.tif (37色相×21明度=777パッチ) をOHPフィルムに印刷
   2. プラチナプリントで露光・現像
   3. プリント結果をスキャン
   4. ColorBlockerプログラムで解析
      - 最も白いパッチ → Maximum Blocker
      - 最も線形的な階調 → Optimum Blocker
   ```

4. **重要な発見: インクジェットのカラー調合**
   
   **ColorBlockerの本質的価値:**
   - インクジェットプリンターは RGB値を CMY(K)インクの「調合」で再現
   - 各色相で異なるインクの組み合わせ:
     - 赤系: Magenta + Yellow
     - シアン系: Cyan多め
     - 緑系: Cyan + Yellow
   - **各インクの色素によってUV遮断性能が異なる**
   - ColorBlockerは「どのインクの組み合わせが最もUV遮断するか」を実測

5. **Photo Black vs カラーインク**
   
   - **V=0 (RGB 0,0,0)**: Photo Blackインク単独 → 最強のUV遮断
   - **V>0**: CMYカラーインクの調合 → インク重ね合わせ効果
   - ColorBlockerはカラーインク領域を評価している

6. **階調の反転（ネガティブの原理）**
   
   重要な理解:
   ```
   プリントで白を出す → UV遮断 → ネガが濃い → RGB低い
   プリントで黒を出す → UV透過 → ネガが薄い → RGB高い
   ```
   
   EDN_HSB2.tifは既に「ネガティブ」として設計:
   - HSB V=0 (暗) → RGB(0,0,0) (濃い) → プリント白
   - HSB V=240 (明) → RGB(195,195,195) (薄い) → プリント黒

7. **現行システムの制約と改善の余地**
   
   **制約:**
   - 色相解像度: 37色相（10°間隔）
   - 彩度: 実質2段階のみ
   - 明度: 21段階
   
   **改善案:**
   - 360色相（1°間隔）× 33明度 = 1,188パッチ
   - A4サイズ（168×154mm）に収まる設計
   - プラチナプリント1枚で完結（コスト最小）
   - 色相精度36倍向上

### 技術的発見

- `EDN_HSB2.tif`: 印刷用カラーチャート (2895×1711px, 平均輝度96.59)
- `EDN_HSB2_template.tif`: スキャン結果の位置合わせ用テンプレート (平均輝度250.71)
- 測定座標: `(74*hue_idx + 94, 74*bright_idx + 94)` から 40×40px サンプリング
- 輝度計算: `0.2125*R + 0.7154*G + 0.0721*B`
