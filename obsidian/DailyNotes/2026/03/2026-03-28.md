## QTR v14カーブとP700インク使用状況の検証

### 01:19 - インク使用に関する質問対応

**きっかけ:**
- ユーザーからv14カーブでシアン（C）やライトグレー（LGY）インクが減少している旨の指摘
- カーブファイルではKインクのみ指定しているはずなのに、他のインクが使われている可能性

**調査内容:**

1. **v14カーブファイルの検証**
   - Kインク以外の全チャンネル（C, M, Y, LC, LM, LK, LLK, V, MK）が完全に0であることを確認
   - カーブデータは正しく設定されている

2. **P700インク残量の確認**
   - スクリーンショット（2026-03-28 1.32.26）でインク配列を確認
   - Photo Black (PK): 約70%（最も減少）← 正常
   - Light Cyan (LC): 約50%
   - Gray (GY): 約60%
   - Vivid Light Magenta (VLM): 約30-40%
   - 他のインクも若干減少の可能性

3. **チャンネルマッピングの検証**
   ```
   QTRマッピング: K, C, M, Y, LC, LM, LK, LLK, V, MK
   P700実配列:    PK,C, VM,Y, LC, VLM,GY, LGY,  V, MK

   結論: 完全一致 - Kカーブ = Photo Black に正しくマッピング
   ```

**検証結果:**

```bash
# v14カーブの全チャンネル検証結果

C curve:   合計 = 0（全て0）
M curve:   合計 = 0（全て0）
Y curve:   合計 = 0（全て0）
LC curve:  合計 = 0（全て0）
LM curve:  合計 = 0（全て0）
LK curve:  合計 = 0（全て0）
LLK curve: 合計 = 0（全て0）
V curve:   合計 = 0（全て0）
MK curve:  合計 = 0（全て0）
```

**結論:**

- QTRカーブ設定: 正しい（Kのみ使用）
- チャンネルマッピング: 正しい（K → Photo Black）
- 他のインク減少の原因: プリンタードライバーレベルでの自動補正の可能性が高い
  - 高濃度Kインク使用を検知してヘッド保護のため他インクを微量混合
  - 階調改善のための自動処理
  - またはヘッドメンテナンス動作（クリーニング等）
