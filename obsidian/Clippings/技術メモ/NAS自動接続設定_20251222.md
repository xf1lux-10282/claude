### 3. 作成したファイル

#### 自動マウントスクリプト
**ファイルパス:** `~/bin/auto_mount_nas.sh`

**機能:**
- パソコン起動時にNASへ自動接続
- 既にマウントされている場合はスキップ
- ネットワーク接続の確認（最大60秒待機）
- macOSのキーチェーンから認証情報を取得
- 詳細ログの出力

**ログファイル:** `~/Library/Logs/nas_auto_mount.log`

#### LaunchAgent設定
**ファイルパス:** `~/Library/LaunchAgents/com.user.nas.automount.plist`

**設定内容:**
- ラベル: `com.user.nas.automount`
- 起動時実行: 有効（`RunAtLoad: true`）
- 定期チェック: 5分ごと（`StartInterval: 300`）
- 実行スクリプト: `~/bin/auto_mount_nas.sh`

**ログファイル:**
- 標準出力: `~/Library/Logs/nas_automount_stdout.log`
- 標準エラー: `~/Library/Logs/nas_automount_stderr.log`

### 4. 実施した作業手順

1. NAS自動マウントスクリプトの作成
2. スクリプトに実行権限を付与

### 5. トラブルシューティング

**問題:** iCloudドライブ内のスクリプトが実行できない
- エラー: `Operation not permitted`
- 原因: LaunchAgentからiCloudドライブ内のファイルを実行する際のセキュリティ制限
- 解決策: スクリプトをローカルディレクトリ（`~/bin/`）にコピー

### 6. 動作確認結果

- LaunchAgentのステータス: `0`（正常）
- スクリプト実行ログ: NASが既にマウントされていることを確認
- 自動接続機能: 正常に動作中

## 今後の運用

### 起動時の動作
次回パソコン起動時に自動的にNASへ接続されます。

### 定期チェック
5分ごとにNASの接続状態をチェックし、切断されていた場合は自動的に再接続します。

### ログの確認方法

```bash
# メインログ
cat ~/Library/Logs/nas_auto_mount.log

# 標準出力ログ
cat ~/Library/Logs/nas_automount_stdout.log

# エラーログ
cat ~/Library/Logs/nas_automount_stderr.log
```

### LaunchAgentの管理

```bash
# 状態確認
launchctl list | grep nas

# 停止
launchctl unload ~/Library/LaunchAgents/com.user.nas.automount.plist

# 再開
launchctl load ~/Library/LaunchAgents/com.user.nas.automount.plist

# 手動実行
~/bin/auto_mount_nas.sh
```
