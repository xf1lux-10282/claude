# アイデア管理ガイド

外出先・就寝前など、**思いついた瞬間にアプリ（携帯含む）で記録するアイデア**を集める場所です。
後日VS Code等で具現化する起点になります。

## 基本ルール

- **1アイデア＝1ファイル**
- ファイル名: `YYYY-MM-DD_アイデア要旨.md`
  - 例: `2026-06-04_暗室の音をインスタレーション化.md`
  - 日付は必ずシステムカレンダー（`date`）で確認してから付ける
- 思いついた時点では完璧でなくてよい。**まず残すことを優先**する
- 後日、そのファイルを育てて具現化していく

## ステータス（アイデアの状態）

| 記号 | 状態 | 意味 |
|------|------|------|
| 💡 | 思いつき（生） | メモしただけ。未整理 |
| 🌱 | 具体化検討中 | 育てている最中 |
| 🚧 | 着手中 | VS Code等で実作業に入った |
| ✅ | 実現 | 形になった |
| 🗄 | 保留・見送り | 今はやらない（消さずに残す） |

## ファイルのテンプレート

新しいアイデアファイルは以下の形で作成する：

```markdown
---
作成日: YYYY-MM-DD
ステータス: 💡思いつき
記録環境: Claude Code（携帯/このアプリ） / VS Code
関連: [ ]
---

# アイデア要旨（タイトル）

## 思いついたこと
<!-- 生のまま。箇条書きでも文章でもよい -->

## 背景・きっかけ
<!-- なぜ思いついたか、何を見て/感じて -->

## 具現化メモ
<!-- どう形にするか。後日PCで追記していく -->

## 関連ファイル・参考
- [ファイル名](パス)

## 次にやること
- [ ]
```

## 運用の流れ

1. 携帯で思いついたら、このアプリで「○○というアイデアを記録して」と頼む → 1ファイル作成
2. 後日PC（VS Code）で開いて、具現化メモを育てる → ステータスを 🌱→🚧 へ
3. 形になったら ✅、今はやらないものは 🗄 にして残す
4. 締切が絡むアイデアは [[締切タスク_写真家活動]] 等にも登録する
