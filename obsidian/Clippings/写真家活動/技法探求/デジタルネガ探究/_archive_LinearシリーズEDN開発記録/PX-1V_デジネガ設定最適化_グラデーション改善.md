# PX-1V デジネガ設定最適化 - グラデーション改善（EDN方式）

## 目的
PX-1Vでのデジネガ制作において「なだらかなグラデーション変化の表現ができない」問題を解決する

## 使用ワークフロー
**Easy Digital Negatives (EDN)** 方式
- Color Blocker（UV波長特性に基づく最適インク配合）
- Absolute Beginner補正値
- スキャナーによる測定・計算
- 3D LUTおよびLUTによる階調変換

## 問題の原因分析

### 発見された主要な問題

#### 1. **カラーマネジメントの致命的な誤設定** ← 最大の原因
```
【誤った設定】
カラー処理: "プリンターによるカラー管理"（Epson Color Control）
  ↓
プリンター側が全ての色/階調を自動補正
  ↓
EDNで作成した3D LUT + 補正LUTが完全に無効化
  ↓
スキャナーで測定・最適化した値が全て無意味に
```

**これが「なだらかなグラデーションができない」最大の原因**

#### 2. **階調分解能 = ノズル分散の制約**
- 5760×1440 dpiでの超高密度ドット配置
- インクの物理的な重なりが増加
- 微妙な濃度差がインク密度で吸収される（トーンジャンプ）

#### 3. **不適切なプリンター設定**
- Black Enhance Overcoat: ON → さらにインク重なり増加
- 双方向印刷: ON → バンディング（筋）の原因
- スムージング: ON → 階調データを勝手に改変

## 【最初に試すべき設定】正しいプリント設定（EDN方式）

### Photoshop側の設定

#### カラーマネジメント設定（最重要）
```
ファイル > プリント

【カラーマネジメント】
□ カラー処理: "Photoshopによるカラー管理" ★★★最重要★★★
             ^^^^^^^^^^^^^^^^^^^^^^
             （「プリンターによるカラー管理」ではない！）

□ プリンタープロファイル: "一般 RGB プロファイル" ★推奨

   【選択肢がない場合の代替】
   - "EPSON_sRGB"
   - "ドキュメント: Adobe RGB (1998)"

□ マッチング方法: 相対的な色域を維持
□ 黒点の補正: OFF
```

**なぜ「Photoshopによるカラー管理」が絶対必要か**：
- EDNの3D LUTとAbsolute Beginner補正値はPhotoshop側で完結
- プリンター側は「受け取った値をそのまま出力」すべき
- 「プリンターによるカラー管理」を選ぶと、せっかくスキャナーで測定・計算した値が全て破壊される

### PX-1Vドライバー設定

#### 基本設定
```
□ 用紙種類: OHPシート または 超高精細専用フィルム
□ 印刷品質: 1440×1440 dpi ★重要（まず試す）
```

#### カラー設定
```
□ カラーマッチング: ColorSync ★重要
□ カラー処理: 色補正なし ★重要
             ^^^^^^^^
             （Photoshop側で既に処理済み）
```

#### 詳細設定
```
□ 双方向印刷: OFF ★重要
□ Black Enhance Overcoat: OFF ★重要
□ スムージング（文字/輪郭）: OFF ★重要
□ 反射低減: OFF
```

### 設定変更の理由

| 設定項目 | 変更前（誤） | 変更後（正） | 理由 |
|---------|------------|------------|------|
| **カラー処理** | **プリンターによるカラー管理** | **Photoshopによるカラー管理** | **EDNのLUTを有効化（最重要）** |
| プロファイル | 写真用紙 | 一般 RGB プロファイル | ニュートラル、EDNを尊重 |
| 解像度 | 5760×1440 dpi | 1440×1440 dpi | インク重なり減少、濃度の線形性向上 |
| カラーマッチング | Epson Color Control | ColorSync | プリンター側の余計な補正を排除 |
| Black Enhance | ON | OFF | インク重ね減少 |
| 双方向印刷 | ON | OFF | バンディング軽減、位置精度向上 |
| スムージング | ON | OFF | 階調データの正確な再現 |

## 【この設定の問題点と限界】

### 1. 解像度を下げることによるトレードオフ

#### 1440×1440 dpiの特性
- ✅ **改善**: インク重なり減少 → 階調の線形性向上
- ✅ **改善**: トーンジャンプ軽減
- ❌ **懸念**: グラデーションの滑らかさが5760×1440 dpiより劣る可能性
- ❌ **懸念**: エッジの解像感が若干低下する可能性

#### 検証が必要な点
- カラーマネジメント修正後、5760×1440 dpiでも改善するか
- 1440×1440 dpiと5760×1440 dpiのどちらが最適か

### 2. 「一般 RGB プロファイル」の限界

#### 現状
- TPS-100専用のカスタムICCプロファイルは存在しない
- 「一般 RGB プロファイル」= 汎用的でニュートラル
- OHPフィルムの実際の特性（ドットゲイン、濃度特性）が完全には反映されない

#### 今後の課題
- EDN方式で実質的な補正は完了している
- 厳密なICCプロファイルは追加作成可能（分光光度計が必要）

### 3. ピエゾヘッドの物理的制約

#### ノズル配置による限界
- 垂直方向（用紙送り方向）: 1440 dpiがノズル物理配置の限界
- 水平方向: 5760 dpiまで可能だが、垂直方向が制約
- **根本的な階調分解能はノズル密度に依存**

#### ディザリングアルゴリズムの影響
- エプソンドライバーのディザリング処理は非公開
- 中間調でのドット配置パターンは制御不可
- より高度な制御には専用RIPソフトが必要（例：QuadToneRIP）

## 【セッションで得られた重要な知見】

### 1. EDNワークフローが無効化されていた

#### 問題の全体像
```
【あなたの作業】
1. UV露光機の波長特性測定
2. プリンターインクの波長別吸収特性測定
3. スキャナーでテストプリント測定
4. Color Blocker最適化（3D LUT作成）
5. Absolute Beginner補正値計算（LUT作成）
6. Photoshopで両方のLUTを適用（16bit、AdobeRGB）
   ↓
【ここまでは完璧】
━━━━━━━━━━━━━━━━━━━━━━━━━━
【プリント時に全てが崩壊】
   ↓
7. プリント設定: "プリンターによるカラー管理"
   ↓
エプソンドライバーが：
- 「これは写真用紙だろう」と自動判断
- 写真を綺麗に見せるために自動補正
- S字カーブ、コントラスト強調、色補正
- 写真用紙プロファイル適用
   ↓
せっかく計算したLUTが全て無意味に
```

#### EDN公式の推奨設定
Easy Digital Negatives公式サイト（easydigitalnegatives.com）でも：
- **必ず "Photoshopによるカラー管理"** を選択
- プリンターは「ダム端末」として動作すべき
- 追加の補正は一切不要、むしろ有害

### 2. Color Blockerの仕組み

#### UV波長特性に基づく最適化
```
【EDN Color Blocker方式】
1. UV露光機の波長スペクトル測定
   - UVA（315-400nm）
   - UVB（280-315nm）など

2. 各インクのUV吸収特性
   - K（黒）: 全波長で高い吸収
   - C（シアン）: 短波長側（青〜UV域）で強い
   - M（マゼンタ）: 中波長域（緑）で強い
   - Y（イエロー）: UV域では弱い

3. 最適なインク組み合わせ（Color Blocker）を算出
   - RGB値として定義
   - 3D LUTに格納
   - Photoshopで適用

4. プリント時、RGB→CMYK変換
   - この変換がプロファイルに依存
   - 「一般 RGB プロファイル」= ニュートラルな変換
   - プリンター側の余計な補正なし
```

#### なぜ「一般 RGB プロファイル」が適切か
- メーカー依存の補正が入らない
- EDNの3D LUTが意図した通りに機能
- Color BlockerのRGB値が正確にCMYKインク配合に変換される

### 3. トーンジャンプの根本原因

#### 物理的メカニズム
```
【問題の連鎖】
5760×1440 dpi（超高密度）
  ↓
ドット間隔が極めて狭い
  ↓
OHPフィルム（非吸収性）でインクが横に広がる
  ↓
隣接ドット同士が物理的に重なる/混ざる
  ↓
理論値: 30% → 32% → 34%
実際値: 全て35%程度に収束（濃度の非線形性）
  ↓
トーンジャンプとして認識される
```

#### 追加要因
```
Black Enhance Overcoat ON
  ↓
黒インクをさらに重ねる
  ↓
インク総量が増加
  ↓
ドットゲイン（インク広がり）がさらに悪化
  ↓
微細な濃度差が完全に失われる
```

### 4. カラーマネジメントの二重補正問題

#### 階調データの破壊プロセス
```
【元データ】
RAW/TIFF（16bit）
  ↓
【Photoshop】
EDN 3D LUT適用（Color Blocker）
EDN補正LUT適用（Absolute Beginner）
→ UV露光機に最適化された階調特性
  ↓
【ここまでは正しい】
━━━━━━━━━━━━━━━━━━━━━━━
【ここから階調が破壊される】（旧設定）
  ↓
【Epson Color Control】
"写真を綺麗に見せる"自動補正
→ S字カーブ適用
→ コントラスト強調
→ シャドウ/ハイライト圧縮
  ↓
【写真用紙プロファイル】
写真用紙のインク吸収特性に合わせた補正
→ OHPフィルム（非吸収性）とは正反対
→ ドットゲイン特性が全く異なる
  ↓
【結果】
EDNで計算した階調カーブが完全に歪む
→ デジネガとして使用不可能な階調特性
```

#### 写真用紙とOHPフィルムの特性比較

| 特性 | 写真用紙 | TPS-100（OHPフィルム） |
|------|---------|---------------------|
| インク吸収性 | 高い（インク層に吸収） | なし（表面に留まる） |
| ドットゲイン | 吸収により拡散 | 物理的な広がりのみ |
| 光の性質 | 反射光 | 透過光 |
| 濃度特性 | マット/光沢による差 | 透明度による濃度 |
| 最適プロファイル | 用紙ごとに専用 | OHPフィルム専用が必要 |

### 5. 16bitワークフローの重要性

#### ビット深度と階調分解能

**8bit（256階調）の問題**
```
元画像: 256段階
  ↓
LUT変換（階調圧縮/伸張）
  ↓
結果: 実質100-150段階程度
→ トーンジャンプが目視で確認できるレベル
```

**16bit（65536階調）の利点**
```
元画像: 65536段階
  ↓
LUT変換（階調圧縮/伸張）
  ↓
結果: 数千段階を維持
→ 滑らかなグラデーション保持
```

#### 現在の作業環境（16bit、AdobeRGB）の評価
- ✅ **正しい選択**: 16bit作業により階調の余裕を確保
- ✅ **AdobeRGB**: 広色域、EDN Color Blockerに適切
- ✅ **EDN 3D LUT**: UV波長特性に基づく最適化
- ✅ **EDN補正LUT**: スキャナー測定による正確な補正
- ⚠️ **プリント時の維持**: Photoshopプリント時も16bit出力を維持すること

### 6. 双方向印刷の問題

#### デジネガでの致命的影響

**双方向印刷のメカニズム**
```
→ 右方向の印刷パス
←  左方向の印刷パス（往復）
→ 右方向の印刷パス
```

**問題点**
1. **位置ずれ**
   - ヘッドの機械的精度により、往路と復路で微妙にずれる
   - 高精細（1440-5760 dpi）では数ドット分のずれが致命的

2. **バンディング（縞模様）**
   - 往路と復路でインク吐出量が微妙に異なる
   - グラデーション部分で横縞として現れる

3. **階調の不連続**
   - 位置ずれによりドットパターンが乱れる
   - EDNで計算した階調カーブから逸脱

**単方向印刷のメリット**
- ✅ 位置精度の向上
- ✅ バンディング完全回避
- ✅ 階調の正確な再現
- ❌ 印刷時間が約2倍（許容すべきトレードオフ）

### 7. スムージング機能の階調破壊

#### スムージングの本来の目的
- 文字や図形の輪郭を滑らかに見せる
- ジャギー（ギザギザ）を目立たなくする
- **ビジネス文書や図形向けの機能**

#### デジネガでの悪影響
```
【処理内容】
エッジ検出
  ↓
輪郭周辺のピクセルを補間
  ↓
中間調を自動生成
```

**問題**
- グラデーション部分も「エッジ」として誤認識される可能性
- 階調データが勝手に補間・改変される
- **EDNのLUTで設定した階調カーブが無効化される**

## テストプリント推奨手順

### 1. グラデーションテストパターン作成

#### Photoshopでの作成
```
1. 新規ドキュメント
   - モード: RGBカラー（AdobeRGB）
   - ビット深度: 16bit/チャンネル
   - サイズ: A4横（297×210mm）
   - 解像度: 300-360 dpi

2. グレースケールグラデーション作成
   - グラデーションツール: 黒(R=0,G=0,B=0) → 白(R=255,G=255,B=255)
   - 幅: 280mm程度

3. ステップウェッジ追加
   - 10%刻みの段階的パッチ（K=0%, 10%, 20%...100%）
   - 各パッチ: 20×20mm程度

4. EDN LUT適用
   - 3D LUT（Color Blocker）
   - 補正LUT（Absolute Beginner）
```

## 今後の最適化方向性

### 短期的改善（すぐに実施可能）

1. **設定変更とテストプリント** ← 最優先
   - カラー処理を「Photoshopによるカラー管理」に変更
   - プロファイルを「一般 RGB プロファイル」に設定
   - **この変更だけで劇的に改善する可能性が最も高い**

2. **解像度の最適化**
   - テストB（1440×1440 dpi）とテストC（5760×1440 dpi）の比較
   - カラーマネジメント修正後は、5760×1440 dpiでも良好な結果が得られる可能性

3. **EDN LUTの微調整**（必要に応じて）
   - 新しいプリント設定での出力特性に合わせる
   - EDN公式サイトの手順で再測定・再計算
   - 特に中間調の階調カーブを最適化

### 中期的改善（時間と投資が必要）

1. **インク量リミットの最適化**
   - EDN方式でも設定可能
   - インク総量を80-90%程度に制限
   - インク重なりを抑制し、階調の線形性向上

2. **代替プロファイルのテスト**
   - 他の汎用RGBプロファイルを試す
   - sRGB、AdobeRGBなどとの比較
   - TPS-100に最適なプロファイルを特定

### 長期的改善（本格的な投資）

1. **専用RIPソフトの導入**
   - QuadToneRIP（無料/オープンソース）
   - または商用RIP（例：EFI Fiery）
   - ディザリングアルゴリズムの完全制御
   - より高度な階調管理
   - EDNとの併用も可能

2. **代替メディアの検討**
   - Pictorico OHP Premium（より高品質版）
   - または他社のデジネガ専用フィルム
   - インク特性との相性確認
