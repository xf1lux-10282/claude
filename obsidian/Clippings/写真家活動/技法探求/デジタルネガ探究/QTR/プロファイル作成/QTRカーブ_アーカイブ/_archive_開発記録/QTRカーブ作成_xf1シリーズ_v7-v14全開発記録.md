**標準配合**:

| 成分 | 濃度 | 使用量（1回分） | 役割 |
|------|------|----------------|------|
| シュウ酸第二鉄 (Fe(C₂O₄)₃) | - | 0.60 cc | 鉄塩（酸化剤） |
| パラジウム溶液 (PdCl₂系) | - | 0.675 cc | パラジウム供給源 |
| 塩化白金酸ナトリウム (Na₂PtCl₆) | 20% w/v | 0.25 cc | プラチナ供給源 |

**モル比計算**:

塩化白金酸ナトリウム 20% w/v = 200 g/L
- 分子量 (Na₂PtCl₆): 561.9 g/mol
- モル濃度: 200 / 561.9 ≈ 0.356 M
- 使用量: 0.25 cc → 約 0.089 mmol Pt

パラジウム溶液（PdCl₂ 5% w/v と仮定）:
- 分子量 (PdCl₂): 177.3 g/mol
- モル濃度: 50 / 177.3 ≈ 0.282 M
- 使用量: 0.675 cc → 約 0.190 mmol Pd

**計算されたモル比**: Pt : Pd ≈ 1 : 2.1（パラジウム過剰）

**合計塗布量**:
- 約 1.525 cc / 1回分
- ガラス棒またはホーク（刷毛）で均一に塗布

**注**: パラジウム溶液の正確な濃度により比率は変動します

## 🎯 xf1シリーズの基本設計思想

### 1. バンディング完全除去

**課題**: 従来のPCHIP補間やアンカーポイント方式では、K値の不均等な変化により境界線（バンディング）が発生

**解決策**: **線形補間（np.linspace）による完全均一なGradient分布**

```python
# xf1シリーズの基本生成式
import numpy as np

K_MAX = 33051  # v7-NA の場合
all_inputs = np.arange(256)
k_values = np.linspace(0, K_MAX, 256)
k_values_final = np.round(k_values).astype(int)
```

**結果**:
- v7-NA: Gradient 129-130 K/step（完全均一）
- v5-HC: Gradient 146 K/step（完全均一）
- 全階調でバンディング完全消失

## 🔬 K_max値の決定根拠

### 実測データに基づく最適化

**xf1-v5（HC前身）の実測結果**:

```csv
Input 240: K = 35117 → ネガ濃度 1.08D → プリント紙白到達 ✓
Input 255: K = 37312 → ネガ濃度 1.15D → 過剰（繊維で視認不可）
```

**結論**:
- **v7-NA**: Input 255で紙白（1.08D）到達を目指す → K_max = 33051
  - 計算: 35117 × (240/255) ≈ 33051
  - 狙い: 全Input範囲（0-255）を有効活用

- **v5-HC**: Input 240で紙白（1.08D）到達を維持 → K_max = 37312
  - Input 255は紙白を超える領域として残す
  - 狙い: より急峻なカーブでコントラスト向上

## 🧪 紙白到達の物理的制約

### コットン紙における限界

**観測結果**:
- Input 240-255の階調差がプリント上でほぼ視認不可
- Input 248以降は繊維のテクスチャに埋もれる

**原因**:
1. **コットン繊維の物理的構造**: 表面のランダムな凹凸
2. **プラチナ粒子の沈着特性**: 超高濃度域での飽和
3. **反射光の散乱**: 紙白付近での光学的限界

**結論**:
- これはQTRカーブの問題ではなく、**物理的な材料特性**
- Input 240付近が実用的な紙白到達点
- v7-NA/v5-HCともにこの限界を適切に考慮

## 📐 QTRファイル形式の仕様

### 10チャンネル完全定義の必要性

**QTR .quadファイルの要件**:
```
必須チャンネル: K, C, M, Y, LC, LM, LK, LLK, V, MK（計10チャンネル）
各チャンネル: 256ポイント（Input 0-255）
K以外: 0で埋める（モノクロ印刷のため）
```

**不完全定義の問題**:
- 過去のv1-v3でK、C、M、Yのみ定義 → QTR処理エラー
- 測定値が理論値と乖離（例: 同K値で異なる濃度）

**xf1シリーズの対応**:
```python
# K curve（実際のカーブ）
for k in k_values_final:
    quad_content += f"{k}\n"

# 残り9チャンネル（すべて0）
for channel in ['C', 'M', 'Y', 'LC', 'LM', 'LK', 'LLK', 'V', 'MK']:
    quad_content += f"\n# {channel} curve\n"
    for i in range(256):
        quad_content += "0\n"
```

## 🎨 コントラストの制御原理

### カーブの傾きとコントラストの関係

**数学的関係**:
```
コントラスト ∝ dK/dInput（カーブの傾き）

同じネガ濃度を得るために必要なInput値:
- v7-NA: より高いInput値が必要（緩やかなカーブ）
- v5-HC: より低いInput値で達成（急峻なカーブ）
```

**実例**:

| 目標濃度 | v7-NA必要Input | v5-HC必要Input | 差分 |
|---------|---------------|---------------|------|
| 0.31D | 約72 | 約72 | 同等 |
| 0.52D | 約128 | 約128 | 同等 |
| 0.81D | 約192 | 約192 | 同等 |
| 1.05D | 約240 | 約240 | 同等 |

**注**: 実際のコントラスト差は、K_max の違いによる全体的な傾き（Gradient）の差から生じる

**視覚的効果**:
- **v7-NA（Gradient 129）**: 豊かな中間調、滑らかな階調推移
- **v5-HC（Gradient 146）**: シャープなコントラスト、メリハリある表現

## 🛠️ カーブ生成の実装コード

### xf1-v7-NA生成スクリプト

```python
#!/usr/bin/env python3
"""
PX1V-PtPd-xf1-v7-NA QTRカーブ生成
"""
import numpy as np

# 設定
K_MAX = 33051  # Input 255で紙白（1.08D）到達

# 線形補間で256ポイント生成
all_inputs = np.arange(256)
k_values = np.linspace(0, K_MAX, 256)
k_values_final = np.round(k_values).astype(int)
k_values_final = np.clip(k_values_final, 0, None)

# 検証
gradients = np.diff(k_values_final)
print(f"K値範囲: {k_values_final[0]} - {k_values_final[255]}")
print(f"Gradient: min={gradients.min()}, max={gradients.max()}, mean={gradients.mean():.1f}")

# 単調性チェック
is_monotonic = np.all(np.diff(k_values_final) >= 0)
assert is_monotonic, "カーブが単調増加していません"

# .quadファイル生成
quad_content = """## QuadToneRIP K,C,M,Y,LC,LM,LK,LLK,V,MK
# QuadProfile Version 2.8.0
# PX1V-PtPd-xf1-v7-NA - Natural (Standard Curve)
# - K_max = 33051（Input 255で紙白到達）
# - Gradient均一化（129-130/step）
# - 全10チャンネル定義
# - 実用域: Input 0-255（全範囲）
# Generated: 2026-03-21

# 2026 Daisuke Kinoshita
#
# K curve
"""

for k in k_values_final:
    quad_content += f"{k}\n"

# 残り9チャンネル
for channel in ['C', 'M', 'Y', 'LC', 'LM', 'LK', 'LLK', 'V', 'MK']:
    quad_content += f"\n# {channel} curve\n"
    for i in range(256):
        quad_content += "0\n"

# 保存
with open('/tmp/PX1V-PtPd-xf1-v7-NA.quad', 'w') as f:
    f.write(quad_content)

print("✓ カーブ生成完了")
```

## ⚠️ 重要な注意事項

### カーブ使用時の注意

1. **露光時間の重要性**
   - xf1カーブは**7分露光を基準**に設計
   - 露光時間変更時は必ずテストプリント実施
   - ±30秒の変化でも結果に影響

2. **薬品調合の精度**
   - モル比 Pt:Pd = 1:1 を維持
   - 滴数を正確にカウント
   - 古い薬品は使用しない（劣化の可能性）

3. **環境条件の管理**

### 重要な発見: 露光時間の延長

**v14で露光時間が延長された理由**:

1. **ミッドトーンの明度化の影響**
   - Zone 2勾配を緩くした（117.4 → 115.0 K/step）
   - K-valueが低い = ネガが薄い = 光透過量が多い
   - 結果として、同じ露光時間では紙白到達に必要な光量が不足

2. **Input 240の微細な影響**
   - v12: 29500
   - v14: 29400 (-0.34%)
   - わずかな差でも、ハイライト域では影響が大きい

3. **+3.8%という絶妙なバランス**
   - ミッドトーンの明度化効果を維持しつつ
   - 紙白到達を確実にする

**原則**:
> カーブの勾配を緩くする（K-valueを下げる）場合、
> 露光時間の延長が必要になることがある
> v14: -2.0%勾配緩和 → +3.8%露光延長
