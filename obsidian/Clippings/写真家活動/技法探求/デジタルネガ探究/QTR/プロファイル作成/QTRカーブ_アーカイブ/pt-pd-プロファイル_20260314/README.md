# PX1V-PtPd-Linear v18 - FINAL VERSION

## 概要

Platinum/Palladium Contact Printing用QuadToneRIPカーブの最終最適化版。
v14とv17の最良部分を統合し、全範囲で滑らかなグラデーションを実現。

**作成日**: 2026年3月14日
**MD5ハッシュ**: `505771e0673545816f2b2b4d690536a0`

---

## v18の特徴

### 統合された最適化範囲

1. **Input 48-104（濃度0.26-0.50）**: v14のスプライン補間
   - 制御点: 40, 48, 104, 112
   - 濃度0.25-0.47範囲の境界線を完全解消

2. **Input 112-192（濃度0.55-0.93）**: v17のスプライン補間
   - 制御点: 104, 112, 144, 192, 200
   - 濃度0.55-0.93範囲の境界線を完全解消

3. **その他の範囲**: v12実測値を使用

### 計算式

- **Baseline計算**: `Density = 0.007387 × Baseline + 0.0935`
- **Quad値計算**: `quad_value = int(round(baseline × 257))`
- **目標濃度**: Input 255で1.22（維持）

## 開発の経緯

### v12（ベースライン）
- Input 255で濃度1.22を達成
- 7箇所で勾配 > 0.006（境界線問題）

### v13（第一段階）
- Input 80-96のスプライン補間
- 67.4%の改善
- **問題**: 他の範囲は未改善

### v14（第二段階）
- Input 48-104のスプライン補間
- 77%の改善、境界線0箇所
- ユーザー確認: "かなり滑らかになった"
- **問題**: Input 112-192は未改善

### v15（失敗）
- Input 8-240の全体補間
- **問題**: 過度な補正で濃度が薄くなりすぎ
- ユーザー評価: "本来印刷されるべき点が薄くなりすぎ"

### v17（第三段階、計算式エラー）
- v13ベース + Input 112-192追加
- **問題**: 誤った計算式（baseline × 256 / 30）を使用
- quad値が30倍小さい（Input 88: 375 vs 正しくは11384）

### v17 FIXED（修正版、部分的問題）
- v17の計算式を修正（baseline × 257）
- **問題**: v13使用により Input 80-96が悪化
- ユーザー報告: "濃度0.25-0.47で2本の線が見える"

### v18（最終版）✓
- v14（Input 48-104）+ v17（Input 112-192）の統合
- すべての境界線問題を解消
- ユーザー評価: "とても綺麗なネガができました"

- **目的**: プラチナ/パラジウムプリント用デジタルネガ
