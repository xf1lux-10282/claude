## 🚨 発見した問題

### リニアリティ検証グラフと補正カーブで異なるデータが必要

**ユーザー指摘**:
> リニアリティ検証のグラフ表示用のデータと補正カーブで表示すべきデータが違いますよね？
> リニアリティ検証のグラフがおかしくなっています。

**問題の詳細**:

| グラフ種類 | 目的 | X軸 | Y軸 | データ処理 |
|-----------|------|-----|-----|-----------|
| **リニアリティ検証グラフ** | 測定値が線形か確認 | ステップ番号（0-255） | 測定RGB値 | **RGB値をそのまま正規化（反転なし）** |
| **補正カーブ** | 濃度を線形化するLUT生成 | 入力デジタル値 | 補正後デジタル値 | **RGB値を濃度に変換（反転あり）** |

**間違っていた実装**:
- `normalizeMeasurements()`でRGB値を濃度に変換（反転済み）
- リニアリティ検証グラフでもこの反転済みデータを使用
- → グラフが逆転して表示される

## ✅ 修正内容

### 修正1: `normalizeRgbForGraph()` 関数を追加

**場所**: [core.js:372-388](file:///Users/daisukekinoshita/Library/Mobile%20Documents/iCloud~md~obsidian/Documents/precision-edn/js/core.js#L372-L388)

```javascript
/**
 * リニアリティ検証グラフ用: RGB値をそのまま正規化（反転なし）
 * @param {Array} measurements - RGB値の配列
 * @returns {Array} 正規化された値の配列（0-1）
 */
normalizeRgbForGraph(measurements) {
  // EDN RGB256チャートはグレースケール（R=G=B）なので、R値を直接使用
  const values = measurements.map(rgb => rgb[0]);  // R値のみ使用

  const min = Math.min(...values);
  const max = Math.max(...values);

  // RGB値をそのまま正規化（反転なし）
  // RGB最小（最暗） → 0
  // RGB最大（最明） → 1
  return values.map(v => (v - min) / (max - min));
}
```

### 修正2: `normalizeMeasurements()` のコメントを明確化

**場所**: [core.js:390-406](file:///Users/daisukekinoshita/Library/Mobile%20Documents/iCloud~md~obsidian/Documents/precision-edn/js/core.js#L390-L406)

```javascript
/**
 * 補正カーブ用: 測定値を濃度として正規化（RGB値を反転）
 * @param {Array} measurements - RGB値の配列
 * @returns {Array} 正規化された濃度の配列（0-1）
 */
normalizeMeasurements(measurements) {
  // EDN RGB256チャートはグレースケール（R=G=B）なので、R値を直接使用
  const values = measurements.map(rgb => rgb[0]);  // R値のみ使用

  const min = Math.min(...values);  // RGB最小 = 最暗（高濃度）
  const max = Math.max(...values);  // RGB最大 = 最明（低濃度）

  // 濃度として正規化（RGB値を反転）
  // RGB最大（最明） → 濃度0（低濃度）
  // RGB最小（最暗） → 濃度1（高濃度）
  return values.map(v => (max - v) / (max - min));
}
```

## 🔍 データフローの整理

```
測定RGB値（スキャン）
  ↓
  ├→ normalizeRgbForGraph()
  │   └→ RGB値をそのまま正規化（反転なし）
  │       └→ リニアリティ検証グラフで表示
  │
  └→ normalizeMeasurements()
      └→ RGB値を濃度に変換（反転あり）
          └→ 補正カーブ生成に使用
```

### 具体例

測定結果:
- ステップ0: RGB=5（最暗）
- ステップ128: RGB=128（中間）
- ステップ255: RGB=250（最明）

**`normalizeRgbForGraph()`（反転なし）**:
```
正規化[0]   = (5 - 5) / (250 - 5) = 0 / 245 = 0
正規化[128] = (128 - 5) / (250 - 5) = 123 / 245 ≈ 0.502
正規化[255] = (250 - 5) / (250 - 5) = 245 / 245 = 1
```
→ リニアリティ検証グラフ: 0 → 0.502 → 1（右上がり）

**`normalizeMeasurements()`（反転あり）**:
```
濃度[0]   = (250 - 5) / (250 - 5) = 245 / 245 = 1（高濃度）
濃度[128] = (250 - 128) / (250 - 5) = 122 / 245 ≈ 0.498
濃度[255] = (250 - 250) / (250 - 5) = 0 / 245 = 0（低濃度）
```
→ 補正カーブ: 濃度1 → 濃度0.498 → 濃度0
