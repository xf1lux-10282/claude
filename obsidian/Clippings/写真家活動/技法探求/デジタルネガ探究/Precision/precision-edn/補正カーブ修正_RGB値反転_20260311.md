# Precision EDN - 補正カーブ修正（RGB値反転）

**作成日**: 2026-03-11

**重要度**: 🔴 高（補正カーブの根本的な誤り）

## 🚨 発見した重大な問題

### RGB値と濃度の関係が逆転していた

**問題**:
`normalizeMeasurements()` 関数でRGB値をそのまま正規化していたため、濃度の概念が逆になっていました。

**間違っていた実装**:
```javascript
normalizeMeasurements(measurements) {
  const values = measurements.map(rgb => rgb[0]);  // R値のみ使用
  const min = Math.min(...values);
  const max = Math.max(...values);

  // 0-1 に正規化
  return values.map(v => (v - min) / (max - min));
  // → RGB最小（最暗） = 0
  // → RGB最大（最明） = 1
}
```

**なぜ間違っていたか**:

| RGB値 | 見た目 | 間違った正規化 | 正しい濃度 |
|-------|--------|----------------|-----------|
| RGB=250（最大） | 明るい（紙の白） | 1 | **0**（低濃度） |
| RGB=5（最小） | 暗い（インクの黒） | 0 | **1**（高濃度） |

**影響**:
- 補正カーブが逆向きになる
- 暗い部分を明るくする補正が、明るい部分を暗くする補正になってしまう
- LUTが逆転し、プリント結果が期待と真逆になる

## ✅ 修正内容

### [core.js:377-388](file:///Users/daisukekinoshita/Library/Mobile%20Documents/iCloud~md~obsidian/Documents/precision-edn/js/core.js#L377-L388)

**修正後**:
```javascript
normalizeMeasurements(measurements) {
  // EDN RGB256チャートはグレースケール（R=G=B）なので、R値を直接使用
  const values = measurements.map(rgb => rgb[0]);  // R値のみ使用

  const min = Math.min(...values);  // RGB最小 = 最暗（高濃度）
  const max = Math.max(...values);  // RGB最大 = 最明（低濃度）

  // 濃度として正規化（RGB値を反転）
  // RGB最大（最明） → 濃度0（低濃度）
  // RGB最小（最暗） → 濃度1（高濃度）
  return values.map(v => (max - v) / (max - min));
}
```

**変更点**:
- `(v - min) / (max - min)` → `(max - v) / (max - min)` に変更
- RGB値を反転して濃度に変換

**数式の意味**:
```javascript
// 変更前: (v - min) / (max - min)
// RGB=250（最明） → (250 - 5) / (250 - 5) = 245/245 = 1 ← 間違い
// RGB=5（最暗） → (5 - 5) / (250 - 5) = 0/245 = 0 ← 間違い

// 変更後: (max - v) / (max - min)
// RGB=250（最明） → (250 - 250) / (250 - 5) = 0/245 = 0 ✓ 低濃度
// RGB=5（最暗） → (250 - 5) / (250 - 5) = 245/245 = 1 ✓ 高濃度
```

## 📊 補正カーブの正しい理解

### EDN方式の濃度概念

1. **濃度0（低濃度）**: 紙の白、光を多く反射 → RGB値が大きい
2. **濃度1（高濃度）**: インクの黒、光を少ししか反射 → RGB値が小さい

### デジタルネガ印刷の流れ

```
デジタル値0   → ネガで最暗 → 紫外線透過少 → プリントで最明（紙の白） → RGB=250 → 濃度0
デジタル値128 → ネガで中間 → 紫外線透過中 → プリントで中間（グレー） → RGB=128 → 濃度0.5
デジタル値255 → ネガで最明 → 紫外線透過多 → プリントで最暗（インクの黒）→ RGB=5   → 濃度1
```

### 補正カーブの目的

**問題**: デジタル値が均等でも、プリント濃度が不均等になる

例：
```
デジタル値0   → RGB=250 → 濃度0
デジタル値128 → RGB=180 → 濃度0.29（本来0.5であるべき）← 明るすぎる
デジタル値255 → RGB=5   → 濃度1
```

**補正**: デジタル値を調整して、濃度を線形化する

```
目標濃度0.5を得るには？
→ 測定データから「RGB=128（濃度0.5）を得るにはデジタル値190が必要」と判明
→ LUT[128] = 190
→ デジタル値128を入力すると、内部では190に変換 → RGB=128（濃度0.5）を実現
```

### 修正前（間違い）

```javascript
// RGB=250（最明）→ 1、RGB=5（最暗）→ 0

// 理想値（ステップ番号の正規化）
ideal = [0, 0.004, 0.008, ..., 0.992, 0.996, 1]

// 逆カーブ
// 「正規化値1を得るには、デジタル値0を使う」← 逆！
```

**結果**: 補正カーブが逆向きになり、暗い部分がさらに暗くなる

### 修正後（正しい）

```javascript
// RGB=250（最明）→ 0、RGB=5（最暗）→ 1
normalized = [0, 0.02, 0.05, ..., 0.95, 0.98, 1]

// 理想値（ステップ番号の正規化）
ideal = [0, 0.004, 0.008, ..., 0.992, 0.996, 1]

// 逆カーブ
// 「濃度0.5を得るには、デジタル値Yを使う」← 正しい！
```

**結果**: 補正カーブが正しい向きになり、濃度が線形化される
