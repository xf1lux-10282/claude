### 2. 「理想値」の定義が曖昧

**場所**: [core.js:406-410](file:///Users/daisukekinoshita/Library/Mobile%20Documents/iCloud~md~obsidian/Documents/precision-edn/js/core.js#L406-L410)

```javascript
// 理想値（線形）
const ideal = [];
for (let i = 0; i < normalized.length; i++) {
  ideal.push(i / (normalized.length - 1));
}
```

**現在の実装**:
- `ideal[i] = i / (length - 1)`
- これは「ステップ番号の正規化」（0, 0.004, 0.008, ..., 1）

**EDN方式の正しい理解**:
- ステップ番号（0-255）= **入力デジタル値**
- スキャンしたRGB値 = **実際の出力濃度**
- 補正カーブ = **線形な濃度勾配を実現するための変換**

**例**:
- ステップ0（デジタル値0）→ スキャン結果R=5（用紙の白）
- ステップ128（デジタル値128）→ スキャン結果R=150（明るすぎる）
- ステップ255（デジタル値255）→ スキャン結果R=240（インクの黒）

**理想的な線形勾配**:
- ステップ0 → R=5（最明）
- ステップ128 → R=122.5（中間、(5+240)/2）
- ステップ255 → R=240（最暗）

**補正カーブの目的**:
- 入力: ステップ128（デジタル値128）
- 現状: R=150が出力される
- 目標: R=122.5を出力したい
- 補正: デジタル値を110に下げる → `LUT[128] = 110`

**現在のコードの問題**:
- `ideal[i] = i / (length - 1)` は、単なる正規化
- これは「ステップ番号が均等」という意味であり、「濃度が線形」という意味ではない

## 📊 EDN方式の正しい補正カーブとは

### EDN方式の前提

1. **入力**: デジタル値（0-255、または0-65535）
2. **プロセス**: デジタルネガ → プリント → スキャン
3. **測定**: スキャンしたRGB値（実際の濃度）
4. **目標**: **線形な濃度勾配**を実現

### 線形な濃度勾配とは

デジタル値が均等に変化すると、濃度も均等に変化する状態：

```
デジタル値    理想的な濃度    実際の測定濃度
0          →  0.0          →  0.02（用紙の白、わずかにグレー）
1          →  0.004        →  0.015
2          →  0.008        →  0.02
...
128        →  0.5          →  0.6（明るすぎる）
...
255        →  1.0          →  0.95（インクの黒、完全には暗くならない）
```

**問題**: デジタル値128で濃度0.6が得られるが、理想は0.5

**補正**: デジタル値を下げて、濃度0.5を実現する
- 測定データから「濃度0.5を得るにはデジタル値110が必要」と判明
- 補正LUT: `LUT[128] = 110`

### 現在の実装の問題

```javascript
// 理想値（線形）
const ideal = [];
for (let i = 0; i < normalized.length; i++) {
  ideal.push(i / (normalized.length - 1));  // 0, 0.004, 0.008, ..., 1
}
```

これは「ステップ番号の正規化」であり、**デジタル値の正規化**としては正しいです。

**しかし**、問題は以下の点：
- `measured[i]` は「ステップiで測定した濃度」（正規化、0-1）
- `ideal[i]` は「ステップiの正規化されたステップ番号」（0-1）

**ここで混同が起きている**:
- `ideal[i]` は「ステップiの理想的な濃度」であるべき
- 現在は「ステップiのデジタル値」になっている

### 正しい実装

#### オプション1: ステップ番号を濃度として扱う（現在の方式、実は正しい可能性）

**前提**:
- EDN RGB256チャートはステップ0-255
- ステップ番号が均等 = デジタル値が均等
- 目標: デジタル値が均等に変化すると、濃度も均等に変化

**ロジック**:
1. `measured[i]`: ステップiで測定した濃度（正規化、0-1）
2. `ideal[i]`: ステップiの正規化された番号 = `i / 255`
3. 逆関数: 「濃度Xを得るには、デジタル値（ステップ番号）Yを使う」

**例**:
- ステップ0: 測定濃度0.02（最明）
- ステップ128: 測定濃度0.6（中間だが明るすぎる）
- ステップ255: 測定濃度0.95（最暗）

測定濃度をソート:
```
濃度0.02 → ステップ0（正規化: 0/255 = 0）
濃度0.6  → ステップ128（正規化: 128/255 = 0.502）
濃度0.95 → ステップ255（正規化: 255/255 = 1）
```

逆関数を構築（「濃度Xを得るには、ステップYを使う」）:
```
目標濃度0.5を得るには？
→ 測定データから補間: 濃度0.5はステップ110付近で得られる
→ LUT[128] = 110（正規化: 110/255 = 0.431）
```

**これは正しい！**

#### オプション2: 濃度を明示的に線形化する

**前提**:
- 目標: 濃度が0から1まで線形に変化
- 測定濃度の範囲: 0.02（最明）〜 0.95（最暗）

**ロジック**:
1. 測定濃度を0-1に正規化: `(measured - 0.02) / (0.95 - 0.02) = (measured - 0.02) / 0.93`
2. 理想的な濃度: `i / 255`（線形勾配）
3. 逆関数: 「正規化された濃度Xを得るには、デジタル値Yを使う」

**現在の実装はこちらに近い**（`normalizeMeasurements()`で0-1に正規化済み）

## ✅ 検証：現在の実装は正しいか？

### 現在のコードフロー

1. **測定値を正規化**（[core.js:377-386](file:///Users/daisukekinoshita/Library/Mobile%20Documents/iCloud~md~obsidian/Documents/precision-edn/js/core.js#L377-L386)）:

   ```javascript
   normalizeMeasurements(measurements) {
     const values = measurements.map(rgb => rgb[0]);  // R値のみ
     const min = Math.min(...values);
     const max = Math.max(...values);
     return values.map(v => (v - min) / (max - min));  // 0-1に正規化
   }
   ```

2. **理想値を生成**（[core.js:406-410](file:///Users/daisukekinoshita/Library/Mobile%20Documents/iCloud~md~obsidian/Documents/precision-edn/js/core.js#L406-L410)）:

   - `measured[i]`: ステップiの正規化された測定濃度（0-1）
   - `ideal[i]`: ステップiの正規化されたステップ番号（0-1）
   - ペアをソート（measured基準）
   - 逆関数を構築: 「濃度Xを得るには、ステップYを使う」

### 具体例で検証

**前提**:
- EDN RGB256チャート（256ステップ）
- ステップ0: R=5（最明）
- ステップ128: R=180（中間だが明るすぎる）
- ステップ255: R=250（最暗）

**ステップ1: 正規化**
```javascript
min = 5, max = 250
normalized[0] = (5 - 5) / (250 - 5) = 0
normalized[128] = (180 - 5) / (250 - 5) = 175 / 245 ≈ 0.714
normalized[255] = (250 - 5) / (250 - 5) = 1
```

**ステップ2: 理想値**
```javascript
ideal[0] = 0 / 255 = 0
ideal[128] = 128 / 255 ≈ 0.502
ideal[255] = 255 / 255 = 1
```

**ステップ3: ペア作成**
```javascript
pairs = [
  { measured: 0, ideal: 0 },        // ステップ0
  { measured: 0.714, ideal: 0.502 }, // ステップ128
  { measured: 1, ideal: 1 },        // ステップ255
  ...
]
```

ソート（measured基準、すでに昇順なので変化なし）

**ステップ4: 逆関数を構築**

目標濃度0.5を得るには？
```javascript
targetDensity = 0.5

// 0.5は measured[0]=0 と measured[128]=0.714 の間
// 線形補間:
t = (0.5 - 0) / (0.714 - 0) ≈ 0.7
digitalValue = 0 + 0.7 * (0.502 - 0) ≈ 0.351

// つまり、濃度0.5を得るには、デジタル値0.351（ステップ90付近）を使う
```

**補正LUT**:
- 本来ステップ128（デジタル値0.502）を使うべきところ
- 実際には濃度0.714しか得られない
- 濃度0.5を得るにはステップ90（デジタル値0.351）を使う
- `LUT[128] = 90` → **補正後、暗くなる**

**これは正しい！**
