## 🔑 重要な決定事項

### 1. ユーザーインターフェースの選択

**決定**: Streamlit GUI

**理由**:
- 視覚的に分かりやすい
- ドラッグ&ドロップ対応
- リアルタイムグラフ表示
- ワンクリックダウンロード
- 実装が容易

**代替案**:
- Tkinter（ネイティブアプリ）→ 見た目が古風
- Jupyter Notebook → 環境構築が必要
- CLIのみ → 使いにくい

### 2. ドキュメント構成

**方針**: 3層構造

1. **クイックアクセス層**（START_HERE.md）
   - 後日すぐ使える
   - 最小限の情報

2. **詳細ガイド層**（各種GUIDE.md）
   - 機能ごとの詳細
   - トラブルシューティング

3. **理論・設計層**（PHILOSOPHY.md, COMPARISON.md）
   - 深い理解のため
   - 開発の経緯
   - アプローチの違い

**ナビゲーション**: INDEX.md で全体を俯瞰

### 3. Version 1とVersion 2の関係

**重要な認識**:
- Version 1は「失敗」ではなく「第一歩」
- Version 2は「進化」であり「補完」
- 両方に価値があり、状況に応じて使い分け

**VERSION_COMPARISON.md作成の意義**:
- 両バージョンのアプローチを明確化
- 違いの理由を説明
- 使い分けガイドを提供
- 今後の発展への参考
