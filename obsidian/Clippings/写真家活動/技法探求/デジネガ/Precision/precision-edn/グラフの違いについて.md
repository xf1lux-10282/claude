# Precision EDN vs EDN公式サイト - グラフの違い

**作成日**: 2026-03-11
**質問**: なぜ同じデータで2つのグラフが違うのか？

---

## 📊 観察されたグラフの違い

### Precision EDN のグラフ
**ファイル**: スクリーンショット 2026-03-11 1.33.15.png

**特徴**:
- ピンク線：測定値（平均）
- グレー線：理想線（y=x）
- 点線：EDN_RGB_256_T...jpg（参照データ）
- **階段状のパターン**
- 256ステップすべて表示

### EDN公式サイトのグラフ
**ファイル**: スクリーンショット 2026-03-11 1.33.04.png

**特徴**:
- 緑線：リニアライズ結果

- **滑らかな曲線**

## 🔍 違いの原因

### 原因1: 測定順序の間違い ⭐ 主原因

**Precision EDNで階段状になった理由**:

EDN RGB256チャートは **右から左** の配置：
```
右端（値0、最暗） → 左端（値240、最明）
```

しかし、測定を **左から右** で行った：
```
左端（値240） → 右端（値0）
```

**結果**:
- ステップ1に値240のデータが入る（本来は0）
- ステップ2に値224のデータが入る（本来は16）
- ...逆順になる

これが **階段状のパターン** を生む。

---

### 原因2: データ処理の違い

#### Precision EDN の処理

```
入力データ
  ↓
そのままプロット（補正なし）
  ↓
生データグラフ（階段状）
```

**特徴**:
- 測定値をそのまま表示
- ユーザーが正しい順序で入力する必要
- 補間なし、フィルタなし

#### EDN公式サイトの処理

```
チャート画像アップロード
  ↓
画像認識・自動測定
  ↓
正しい順序で読み取り
  ↓
リニアライズ処理（補正）
  ↓
滑らかな曲線グラフ
```

**特徴**:
- 画像から自動的に正しい順序で読み取る
- スプライン補間で滑らかに
- 自動的にリニアライズ（線形化）

---

## 📐 リニアライズとは

### 定義

**リニアライズ（Linearization）**:
非線形なデータを、理想的な直線（y=x）に近づける処理。

### EDN公式サイトのリニアライズ

```
測定データ:        理想線(y=x):
  濃度                濃度
   ↑                   ↑
   │    ／             │  ／
   │  ／               │ ／
   │／                 │／
   └────→ 入力        └────→ 入力
   (非線形)             (線形)

      ↓ リニアライズ処理

補正後:
  濃度
   ↑
   │  ／
   │ ／
   │／
   └────→ 入力
   (線形に近い)
```

### Precision EDN のアプローチ

Precision EDNは **リニアライズ処理をしない**。

理由:
- 生データを忠実に表示
- ユーザーが実際の非線形性を確認できる
- 補正カーブ生成時に透明性が高い

## 🎯 正しい使い方

### Precision EDN

**ステップ1: 正しい順序で測定**

EDN RGB256チャートの場合:
```
1行目: 右端(0) → 左端(240)
2行目: 右端(1) → 左端(241)
...
16行目: 右端(15) → 左端(255)
```

**ステップ2: 測定順序でそのまま入力**

```
0の濃度
16の濃度
32の濃度
...
255の濃度
```

**結果**: 滑らかなグラフが得られる ✓

### EDN公式サイト

**ステップ1: チャート画像をアップロード**

.jpg または .tif ファイル

**ステップ2: 自動処理**

サイトが自動的に：
- 正しい順序で読み取る
- リニアライズ処理
- LUT生成

**結果**: 滑らかな補正カーブが得られる ✓

## 📊 グラフの見方

### 理想的なグラフ（正しい測定順序）

```
濃度
 ↑
 │        ／￣
 │      ／
 │    ／
 │  ／
 │／
 └─────────→ ステップ

特徴:
- 滑らかな曲線
- 単調増加（または減少）
- 異常な跳ねがない
```

### 問題のあるグラフ（測定順序が逆）

```
濃度
 ↑  ┐   ┐   ┐
 │  └─┐ └─┐ └─┐
 │    └─┐  └─┐  └─
 │      └────┘
 └─────────────→ ステップ

特徴:
- 階段状のパターン
- ジグザグ
- 明らかに異常
```

**この場合の対処**:
1. 測定順序を確認
2. EDN_RGB256_測定ガイド.md を参照
3. 正しい順序で再入力

## 🔧 今回の問題の解決

### 問題

デスクトップのEDN_RGB_256_TEMPLATE-PX7Vを読み込んだ時:
- Precision EDN: 階段状グラフ
- EDN公式サイト: 滑らかなグラフ

### 原因

**測定順序が逆だった**

チャートは右→左だが、左→右で測定・入力した。

### 解決策

**正しい測定順序で再測定・再入力**

1. EDN_RGB256_測定ガイド.md を参照
2. 右端（値0）から測定開始
3. 各行を右→左に測定
4. 測定順序でそのまま入力

**結果**: Precision EDNでも滑らかなグラフが得られる ✓

## 📈 2つのアプローチの使い分け

### EDN公式サイトが優れる場合

**用途**:
- 迅速なLUT生成
- 画像から自動測定したい
- 測定ミスを自動補正してほしい

**利点**:
- 画像アップロードだけで完結
- 自動リニアライズ
- ユーザーエラーが少ない

**欠点**:
- 内部処理が見えない
- カスタマイズ不可
- オンライン専用

### Precision EDNが優れる場合

**用途**:
- 詳細な分析が必要
- カスタムパラメータ調整
- オフラインで作業
- 生データを確認したい

**利点**:
- 生データを忠実に表示
- 処理が透明
- オフラインで動作
- カスタマイズ可能

**欠点**:
- 測定順序を正しく守る必要
- 手動入力
- リニアライズは手動

## 💡 ベストプラクティス

### 測定時

1. **測定ガイドを確認**
   - EDN_RGB256_測定ガイド.md

2. **チャート配置を確認**
   - 右端 = 値0（最暗）
   - 左端 = 値240/255（最明）

3. **測定順序を守る**
   - 右 → 左
   - 上 → 下

4. **記録を丁寧に**
   - 測定順序のまま記録
   - 並び替えない

### データ入力時

1. **測定順序のまま入力**
   - 値の大きさで並び替えない
   - ステップ番号は測定順

2. **グラフで確認**
   - 滑らかな曲線？
   - 階段状なら順序を確認

3. **異常値チェック**
   - 極端な跳ねがないか
   - 単調性が保たれているか

## 🎓 まとめ

### グラフが違う理由

1. **測定順序の違い**: 左→右 vs 右→左
2. **処理の違い**: 生データ vs リニアライズ

### 解決方法

**正しい測定順序で測定・入力する**

チャートの配置を確認し、右端（暗）から左端（明）へ測定。

### 2つのツールの関係

- **EDN公式サイト**: 自動化・簡易化
- **Precision EDN**: 詳細分析・カスタマイズ

どちらも価値があり、用途に応じて使い分ける。
