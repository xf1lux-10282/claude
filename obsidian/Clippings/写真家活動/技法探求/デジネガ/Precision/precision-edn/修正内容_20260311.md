**対象**: EDN RGB256チャートの読み込み順序対応

## 🔧 修正内容

### 問題

EDN RGB256チャートは **右→左** の配置だが、左→右で測定してしまうと、グラフが階段状になる問題がありました。

### 解決策

**チャートタイプ選択機能を追加**し、左→右で測定した場合でも自動的に正しい順序に並び替えられるようにしました。

#### 修正2: convertEdnRgb256Order関数を追加（行910-963）

**EDN RGB256チャートの配置ルールに基づいて並び替え**:

```javascript
function convertEdnRgb256Order(data) {
  // 256ステップでない場合は警告
  if (data.length !== 256) {
    console.warn(`EDN RGB256は256ステップですが、${data.length}ステップのデータが入力されました`);
    return data;
  }

  const reorderedData = [];

  // EDN RGB256の配置ルール:
  // 行番号(0-15) + 列番号(0-15) × 16 = 値(0-255)
  for (let row = 0; row < 16; row++) {
    for (let col = 0; col < 16; col++) {
      // 入力データのインデックス（左→右で測定した場合）
      const inputIndex = row * 16 + (15 - col);

      // 正しい値
      const correctValue = row + col * 16;

      // データを再配置
      if (inputIndex < data.length) {
        reorderedData.push({
          step: correctValue + 1,
          density: data[inputIndex].density,
          originalStep: data[inputIndex].step  // デバッグ用
        });
      }
    }
  }

  // 値順にソート
  reorderedData.sort((a, b) => a.step - b.step);

  return reorderedData;
}
```

**アルゴリズム**:

EDN RGB256の配置:
```
1行目: 右(値0)  → 左(値240)   = 列0,1,2...15 → 値0,16,32...240
2行目: 右(値1)  → 左(値241)   = 列0,1,2...15 → 値1,17,33...241
...
16行目: 右(値15) → 左(値255)  = 列0,1,2...15 → 値15,31,47...255
```

左→右で測定した場合:
```
入力順序: ステップ1,2,3...256
実際の値: 240,224,208...0, 241,225,209...1, ...
```

変換後:
```
正しい順序: 値0,1,2...255
対応する濃度: 入力データから正しく取得
```

### 修正の目的

**ユーザーの測定ミスを自動的に補正**し、正しいグラフを生成できるようにする。

1. **柔軟性**: 正しい順序でも、間違った順序でも対応
