baseline = int(round(i * 255))  # ❌ 最大値が65280にしかならない

**正しい記述**:
```python
baseline = int(round(i * 256))  # ✅ 最大値が65536になる

### ❌ エラー3: 「逆相関」関係の説明不足

**問題**: ネガ濃度とプリント濃度の逆相関関係が明示されていない

**必要な追加セクション** (全ファイルの冒頭):

```markdown
## 🔴 最重要概念: ネガ濃度とプリント濃度の逆相関

コンタクトプリントでは、ネガ濃度とプリント濃度は**逆の関係**になります:

### 物理的メカニズム
- **ネガ濃度 ↑ (濃い・黒い)** → 光を遮る → **プリント濃度 ↓ (明るい・白い)**
- **ネガ濃度 ↓ (薄い・透明)** → 光を通す → **プリント濃度 ↑ (暗い・黒い)**

### 具体例
| 元画像 | Input値 | ネガの状態 | プリントの状態 |
|--------|---------|------------|----------------|
| 黒 | 0 | 薄い（透明に近い） | 濃い（黒い） |
| グレー | 128 | 中間濃度 | 中間濃度 |
| 白 | 255 | 濃い（黒い） | 薄い（白い） |

### カーブ作成への影響
- Input 255（画像の白）を最大ネガ濃度にする
- Input 0（画像の黒）を最小ネガ濃度にする
- これにより、プリントで正しい階調が再現される

⚠️ **注意**: この逆相関を理解していないと、カーブが逆になってしまいます
```

### ❌ エラー4: 257の説明不足

**問題**: `quad_value = int(round(baseline * 257))`の257が説明なしで使われている

**必要な追加説明**:

```markdown
## Quad値計算で257を使う理由

QTRの.quadファイル形式では、Kチャンネルは**257ポイント**定義されます（Input 0-256の257個の値）

### 計算式
```python
quad_value = int(round(baseline * 257))
```

### 257の由来
```
257 = 65536 / 256
    = (16-bit最大値 + 1) / (8-bit Input数)
    = 2^16 / 2^8
```

### 具体例
- Input 0 → Baseline 0.0 → quad = 0
- Input 128 → Baseline 0.5 → quad = 128 (= 0.5 * 257 = 128.5 → 129)
- Input 255 → Baseline 1.0 → quad = 257
- **Input 256** → Baseline 1.0 → quad = 257 (最大値)

⚠️ 256ではなく**257**を使用することで、0-65535の16-bit範囲を正しくマッピングできます
```

### ❌ エラー5: Baseline-Density関係式の導出過程が不明

**問題**: `Density = 0.007387 × Baseline + 0.0935`の定数がどう算出されたか不明

**必要な追加セクション**:

```markdown
## Baseline-Density関係式の導出 (v11で確立)

### 測定データ (v10印刷ネガ)
v10カーブで印刷したネガを測定し、以下のデータを取得:

| Input | Baseline | 実測Density |
|-------|----------|-------------|
| 160 | 155.2 | 1.24 |
| 255 | 210.7 | 1.65 |

### 線形回帰計算
**傾き (slope)**:
```
slope = (D2 - D1) / (B2 - B1)
      = (1.65 - 1.24) / (210.7 - 155.2)
      = 0.41 / 55.5
      = 0.007387387...
      ≈ 0.007387
```

**切片 (intercept)**:
```
intercept = D1 - slope × B1
          = 1.24 - 0.007387 × 155.2
          = 1.24 - 1.146262...
          = 0.093737...
          ≈ 0.0935
```

### 最終式
```python
# 順方向（BaselineからDensity算出）
Density = 0.007387 × Baseline + 0.0935

# 逆方向（目標DensityからBaseline算出）
Baseline = (Density - 0.0935) / 0.007387
```

### ⚠️ 重要な注意
この関係式は**PX-1V + ピクトリコOHPフィルム + プラチナ/パラジウム**の組み合わせ専用です。

**他のプリンター・用紙では必ず再測定が必要**:
1. 任意のカーブでネガ印刷
2. 高濃度域（Input 160-255付近）で2点以上測定
3. 各ポイントのBaseline値（.quadファイルから）と実測Density値を記録
4. 上記と同じ線形回帰で新しいslope/interceptを算出

### 検証
v12以降のすべてのカーブでこの式を使用し、Input 255の実測濃度が目標値±0.02以内に収まることを確認済み。
```
