## 🔬 xf1シリーズ開発プロセス概要

### 開発フローサマリー

xf1シリーズは、以下の段階的プロセスを経てv14に到達しました。

```
【Phase 1】紙白基準値の確立（v7-NA）
    ↓
① デフォルトカーブでグラデーションチャート出力
   - UCカーブまたはリニアカーブ使用
   - 0-255のグラデーション印刷
   - Baseline測定データ取得
    ↓
② ネガ透過濃度測定 → プリント紙白到達点を特定
   - Input 0, 8, 16, ..., 255（33点）測定
   - 紙白到達濃度の特定: 約1.08D
   - Input 240 (K=35117) で紙白到達確認
    ↓
③ 紙白到達K_maxの計算と均等補間
   - K_max計算: 35117 × (255/240) ≈ 33051
   - 0-255で線形補間（np.linspace）
   - 勾配: 129.6 K/step（全域均一）
   - 【重要】バンディング完全消失を確認 ✓
    ↓
④ v7-NAで実プリント → 課題発見
   - 紙白到達: 確認 ✓
   - バンディング: 消失 ✓
   - 課題: ミッドトーンが「やや濃く感じる」
   - 結論: 全域均一勾配では階調最適化に限界

【Phase 2】3ゾーン設計への進化（v8-v9）
    ↓
⑤ 3ゾーン設計の導入
   - Zone 1 (Shadow 0-24): 172.8 K/step（強化）
   - Zone 2 (Midtone 24-232): 124.7 K/step（最適化）
   - Zone 3 (Highlight 232-255): 274.7 K/step（圧縮）
   - v9-SM: 実績ベースライン確立
   - 露光時間: 7分 → 6分に短縮
    ↓
⑥ ミッドトーン段階的最適化（v10-v14）
   - v10: Midtone 119.8 K/step (-3.8% vs v9)
   - v12: Midtone 117.4 K/step (-5.8% vs v9) 「今までで一番いい」
   - v14: Midtone 115.0 K/step (-7.8% vs v9) 最終版
   - Zone境界調整: 232 → 240に変更
   - 露光時間微調整: 6.5分 → 6.75分
   - K_max最適化: 33800（v7基準+2.3%）

【到達点】v14単独運用決定
   - 最も豊かな階調（Zone 2勾配 115.0 K/step）
   - バンディング完全消失（標準偏差0.000）
   - 紙白確実保証（K_max=33800）
   - 後処理柔軟性最大化（Photoshopで調整）
```

> v7-NAの実測により確立された基準値

> v9からv14まで: 合計-7.8%を3段階で達成

## 🎯 v14設計思想

### 基本コンセプト

**目標**: 豊かな階調表現、黒を潰さず、白も消さない

**設計方針**:
- 最も緩やかなミッドトーン勾配（115.0 K/step）で最大の階調表現
- 紙白到達確実保証（K_max=33800）
- コントラスト等の調整はPhotoshop側で行う
- 最大の階調情報を保持し、後処理で柔軟に調整可能

### xf1シリーズの核心技術: 均等補間によるバンディング完全消失

**従来カーブの問題**:
- PCHIP補間やアンカーポイント方式では、K値の不均等な変化により境界線（バンディング）が発生
- グラデーション領域で目に見える線（階調の急変）が出現
- プリント品質を大きく損なう

**xf1シリーズの解決策**:
- **線形補間（np.linspace）による完全均一なGradient分布**
- Zone 2（ミッドトーン）での勾配標準偏差: **0.000**（数学的に完璧）
- グラデーション領域で境界線が完全に消失 ✓

**v14での達成**:
```python
# Zone 2 (24-239): 完全な線形補間
target_zone2_grad = 115.0  # K/step
for i in range(1, zone2_steps + 1):
    k = round(zone2_k_start + target_zone2_grad * i)
```

**結果**:
- Input 24-239（215ステップ）で勾配が完全に均一
- バンディング（境界線）完全消失
- 滑らかで自然なグラデーション表現

---

### なぜv14単独運用なのか

**理由**:
1. v14が最も豊かな階調（Zone 2勾配 115.0 K/step = xf1シリーズで最も緩やか）
2. バンディング完全消失（均等補間による標準偏差0.000）
3. Photoshopでコントラスト調整可能なため、最大の階調情報保持が最適
4. v12やv9の効果（ミッドトーン明るめ）はPhotoshopのカーブ調整で再現可能
5. v14から始めれば、後処理で任意の階調バランスに調整可能

**他バージョンとの比較**:
- v12: Zone 2勾配 117.4 K/step（v14より2.0%急）
- v9-SM: Zone 2勾配 124.7 K/step（v14より7.8%急）
- **結論**: v14が最も豊かな階調 + バンディング完全消失 = 最も柔軟な後処理が可能

## 🛠️ カーブ生成方法

### v14生成スクリプト（完全版）

```python
#!/usr/bin/env python3
"""
PX1V-PtPd-xf1-v14 QTRカーブ生成
控えめなミッドトーン最適化 + Input 248濃度向上
"""
import numpy as np

# Zone 1: Shadow (0-24)
zone1_k_values = []
for i in range(0, 25):
    k = round(4148 * i / 24)
    zone1_k_values.append(k)

# Zone 2: Midtone (24-239) - 115.0 K/step均等勾配
zone2_k_start = 4148
target_zone2_grad = 115.0
zone2_steps = 239 - 24  # 215 steps
zone2_k_values = []
for i in range(1, zone2_steps + 1):
    k = round(zone2_k_start + target_zone2_grad * i)
    zone2_k_values.append(k)

# Input 240: 指定値
k_240 = 29400

# Input 241-247: 240→248を線形補間
k_248 = 31900
transition_k_values = []
steps_240_to_248 = 8
for i in range(1, steps_240_to_248):  # 241-247
    k = round(k_240 + (k_248 - k_240) * i / steps_240_to_248)
    transition_k_values.append(k)

# Zone 3: Highlight (248-255) - 線形補間
zone3_k_start = k_248
zone3_k_end = 33800
zone3_steps = 7
zone3_k_values = []
for i in range(1, zone3_steps + 1):
    k = round(zone3_k_start + (zone3_k_end - zone3_k_start) * i / zone3_steps)
    zone3_k_values.append(k)

# 全K-values結合
k_values = zone1_k_values + zone2_k_values + [k_240] + transition_k_values + [k_248] + zone3_k_values

# 検証
assert len(k_values) == 256, f"K-values count: {len(k_values)} (expected 256)"
assert k_values[0] == 0, f"K[0]: {k_values[0]} (expected 0)"
assert k_values[255] == 33800, f"K[255]: {k_values[255]} (expected 33800)"

# 勾配検証
gradients = np.diff(k_values)
zone1_grad = gradients[0:24].mean()
zone2_grad = gradients[24:239].mean()
zone3_grad = gradients[248:255].mean()

print(f"=== v14 勾配検証 ===")
print(f"Zone 1 (0-24):   {zone1_grad:.1f} K/step")
print(f"Zone 2 (24-239): {zone2_grad:.1f} K/step")
print(f"Zone 3 (248-255): {zone3_grad:.1f} K/step")
print(f"勾配比: {zone3_grad / zone2_grad:.2f}")

# .quadファイル生成
quad_content = """## QuadToneRIP K,C,M,Y,LC,LM,LK,LLK,V,MK
# QuadProfile Version 2.8.0
# Title: PX1V-PtPd-xf1-v14
# Printer: EPSON Stylus Pro 3880
# PPD: EPSON Stylus Pro 3880
# - Shadow (0-24): 172.8 K/step
# - Midtone (24-239): 115.0 K/step (smoothest in xf1 series)
# - Input 240: 29400 (slightly lighter than v12)
# - Input 248: 31900 (denser than v12 for highlight)
# - Highlight (248-255): 271.4 K/step
# - K_max: 33800 (paper white guaranteed)
# - Gradient ratio: 2.67 (acceptable range)
# - Exposure time: 6.75 min (6min 45sec)
# - Target: Richest tonal gradation, post-process flexibility
# Generated: 2026-03-24 by Claude Code

# 2026 Daisuke Kinoshita
# BOOST_K=0 - NO BOOST
#
# K curve
"""

for k in k_values:
    quad_content += f"{k}\n"

quad_content += "#\n"

# 残り9チャンネル（全て0）
for channel in ['C', 'M', 'Y', 'LC', 'LM', 'LK', 'LLK', 'V', 'MK']:
    quad_content += f"# {channel} curve\n"
    for _ in range(256):
        quad_content += "0\n"
    quad_content += "#\n"

# 保存
output_path = 'PX1V-PtPd-xf1-v14.quad'
with open(output_path, 'w') as f:
    f.write(quad_content)

print(f"\n✓ v14カーブ生成完了: {output_path}")
```

#### 問題の発見
- **現象**: 和紙でInput 240-255が全て紙白になる（階調消失）
- **原因**: 和紙の薄さと高い透過性により、v14のネガ濃度差が識別不能

```
「プリントで240-255が紙白になる（階調圧縮）」
→ プリント思考: ネガでも「圧縮」すればいい
→ 実装: K値範囲を狭くする（28000-29400）
→ 結果: さらに悪化（階調完全消失）
```

**プリントとネガの逆転関係を忘れた根本的誤り**

#### 正しい理解への転換（v5-v6）

**✓ 正しいアプローチ: ハイライト拡張**
```
「プリントで240-255が紙白になる（階調圧縮）」
→ ネガ思考: ネガでは「拡張」が必要（逆！）
→ 実装: K値範囲を広くする（27000-32000）
→ 結果: 階調表現可能に
```

**デジタルネガの基本原理**:
```
プリント（デジタル画像）:
  明るくしたい → 高いピクセル値（255）
  階調豊かにしたい → ヒストグラム拡張

ネガ（完全に逆）:
  明るいプリントにしたい → 濃いネガ（高いK値）→ 光を遮る
  プリントで階調豊かにしたい → ネガの濃度差を大きく（K値範囲拡張）
```

**2段階の反転**:
```
デジタル画像 → ネガ（反転） → 光 → プリント（反転）
```

### プリントとネガの逆転関係（重要な教訓）

#### なぜ逆になるのか

**ネガ = 中間媒体**:
```
デジタル画像 → ネガ（反転） → 光 → プリント（反転）
              ↑               ↑
              ここで逆転      ここで逆転
```

1. デジタル画像でInput 255（白）
2. → ネガで高いK値（濃い、黒に近い）
3. → 露光で光を遮る
4. → プリントで紙白（反転して白）

#### 実践的な思考法

**常にネガの立場で考える**:
- 「プリントを明るくしたい」→「ネガを濃くする」
- 「プリントを暗くしたい」→「ネガを薄くする」
- 「プリントの階調を豊かにしたい」→「ネガの濃度差を大きくする」
- 「プリントの階調を圧縮したい」→「ネガの濃度差を小さくする」

**今回のケース**:
```
プリントで240-255の階調が失われている
→ ネガ思考: ネガの240-255の濃度差を大きくする
→ 実装: K値範囲を拡張（27000-32000）
→ 結果: プリントで階調表現が可能に
```

### 重要な発見と原則

#### 1. ハイライト濃度問題の解決

**問題の分析**:
```
3分露光でInput 248が濃く見える
→ 紙白に近いはずなのに視認できる
→ ネガの濃度が不足している
→ Zone 3の勾配（345 K/step）では不十分
```

**解決策**:
```
Zone 3勾配を2倍に強化: 345 → 690 K/step
→ Input 241-254のGY値が大幅増加
→ ネガ濃度が増加
→ より確実な紙白到達
```

#### 2. K vs GY/LGYの役割分担

**設計思想**:
- **K channel**: 濃度ベース + 紙白確実到達
  - 全域で安定したベース濃度（40 K/step）
  - Input 255で40000到達

- **GY channel**: 豊かな階調表現
  - Zone別勾配で最適な階調
  - 主要な階調情報を担当

- **LGY channel**: 微細階調補完
  - 10 K/stepで微細なグラデーション
  - ディザリング解消効果

#### 3. インク混合比率の最適化

**2inkと3ink-v2の比較**:

| 項目 | 2ink | 3ink-v2 | 効果 |
|------|------|---------|------|
| K/step | 30 | 40 | 濃度ベース+33% |
| Zone 3勾配 | 300 | 690 | ハイライト+130% |
| LGY追加 | なし | 10 K/step | 微細階調追加 |
| GY[255] | 0 | 5000 | 紙白質感向上 |
| LGY[255] | - | 5000 | 紙白質感向上 |
| 露光時間 | 5:27 | 4:45 | -13% |

#### 4. Zone 3設計の重要性

**従来（v14等）**:
- Zone 3は241-254（14ステップ）
- 主にInput 255への到達が目的
- 勾配: 271.4 K/step

**3ink-v2**:
- Zone 3は241-254（14ステップ）
- ハイライト階調の豊かさが目的
- 勾配: 690 K/step（2.5倍急）
- Input 248-254の階調表現を重視

**原則**:
> ハイライト域で階調が失われる場合、Zone 3の勾配を大幅に強化する
