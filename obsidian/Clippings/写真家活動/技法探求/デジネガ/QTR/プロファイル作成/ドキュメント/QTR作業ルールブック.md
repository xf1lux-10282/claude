### ルール2: カーブ生成時は必ず10チャンネル構造で出力する

**なぜ**: QuadToneRIPは10チャンネル（K, C, M, Y, LC, LM, LK, LLK, V, MK）の定義を必要とします。Kチャンネルのみでは不十分です。

**正しい実装**:
```python
def write_quad_file(quads, filename):
    """Quadファイルを出力（正しい10チャンネル構造）"""

    with open(filename, 'w') as f:
        # ヘッダー
        f.write('## QuadToneRIP K,C,M,Y,LC,LM,LK,LLK,V,MK\n')
        f.write('# QuadProfile Version 2.8.0\n')
        f.write('# PX1V-PtPd-SP1vXX - Description\n')
        f.write('# ... その他のヘッダーコメント ...\n')
        f.write('# BOOST_K=0 - NO BOOST\n')
        f.write('# K curve\n')  # ← 必須: Kチャンネルコメント行

        # チャンネル1: Kカーブ（実際の値）
        for quad in quads:
            f.write(f'{quad}\n')

        # チャンネル2-10: すべて0（各チャンネルにコメント行を追加）
        channels = ['C', 'M', 'Y', 'LC', 'LM', 'LK', 'LLK', 'V', 'MK']
        for channel in channels:
            f.write(f'# {channel} curve\n')  # ← 必須: 各チャンネルのコメント行
            for j in range(256):
                f.write('0\n')
```

### ルール7: バンディングリスクを必ず解析する

**なぜ**: 急激なQuad値の変化（Δ > 200）はバンディング（境界線）の原因となります。

**実装例**:
```python
def analyze_banding_risk(quads, threshold=200):
    """バンディングリスクを解析"""
    risks = []
    for inp in range(1, 256):
        diff = quads[inp] - quads[inp-1]
        if abs(diff) > threshold:
            risks.append((inp, diff))
    return risks
```

**確認項目**:
- 新規リスク箇所が追加されていないか
- リスク箇所がある場合、段階的な削減/増加に変更

## ⚠️ ルール9: K値は0-65535の16bit範囲で生成する【重要】

### なぜ

QTRの.quadファイルにおいて、K値（各チャンネルの値）は**0から65535の16bit整数**で定義されます。0から255の8bit範囲で生成すると、QTRは正しく濃度を表現できず、カーブが完全に間違った方向に働きます。

### 症状

**誤った実装（0-255範囲）の場合**:
- Input 0（黒100%）: K値 = 0
- Input 128（中間調）: K値 = 128
- Input 255（白100%）: K値 = 255

この場合、ネガ濃度の階調が極端に圧縮され、プリントが意図した濃度にならない。

**正しい実装（0-65535範囲）の場合**:
- Input 0（黒100%）: K値 = 0
- Input 128（中間調）: K値 = 13976（約21%）
- Input 255（白100%）: K値 = 40168（約61%）

### 正しい実装

```python
# QTR定数
QTR_MAX_VALUE = 65535  # 16-bit maximum

def negative_density_to_k(neg_density, sp1v21_k_values, sp1v21_neg_density):
    """
    ネガ濃度をK値（0-65535）に変換

    重要: QTRのK値は16bit（0-65535）の範囲
    """
    # SP1v21のK値→濃度マッピングを使用
    sort_idx = np.argsort(sp1v21_neg_density)
    sorted_density = sp1v21_neg_density[sort_idx]
    sorted_k = sp1v21_k_values[sort_idx]

    # 濃度 → K値の補間関数を作成
    f_d_to_k = interpolate.interp1d(
        sorted_density,
        sorted_k,
        kind='cubic',
        fill_value='extrapolate'
    )

    # 新しい濃度からK値を計算
    k_values = f_d_to_k(neg_density)

    # 0-65535の範囲にクリップ
    k_values = np.clip(k_values, 0, QTR_MAX_VALUE).astype(int)

    return k_values
```

### 誤った実装例（避けるべき）

```python
# ❌ 間違い: 0-255の範囲で生成
def negative_density_to_k_WRONG(neg_density):
    # Input値（0-255）をそのままK値として使用
    k_values = np.clip(target_input, 0, 255)  # ← 間違い！
    return k_values
```

## ⚠️ ルール11: デジタルネガではプリントと逆の発想が必要【重要な教訓】

**作成日**: 2026年4月1日
**教訓元**: xf1-v14-washi-v6開発（v1-v3の失敗とv5-v6での成功）

### なぜ

デジタルネガはデジタル画像（プリント）と**完全に逆の関係**にあります。プリント思考でネガを設計すると、結果が真逆になります。

### プリントとネガの逆転関係

#### プリント（デジタル画像）での考え方
```
明るくしたい → 高いピクセル値（255に近い）
暗くしたい   → 低いピクセル値（0に近い）

階調を豊かにしたい → ヒストグラムを広げる
階調を圧縮したい   → ヒストグラムを狭める
```

#### ネガでの考え方（完全に逆）
```
明るいプリントにしたい → 濃いネガ（高いK値）→ 光を遮る
暗いプリントにしたい   → 薄いネガ（低いK値）→ 光が通る

プリントで階調を豊かにしたい → ネガの濃度差を大きく（K値範囲を広く）
プリントで階調を圧縮したい   → ネガの濃度差を小さく（K値範囲を狭く）
```

### なぜ逆になるのか

**ネガ = 中間媒体 = 2段階の反転**:
```
デジタル画像 → ネガ（反転） → 光 → プリント（反転）
              ↑               ↑
              ここで逆転      ここで逆転
```

1. デジタル画像でInput 255（白）
2. → ネガで高いK値（濃い、黒に近い）
3. → 露光で光を遮る
4. → プリントで紙白（反転して白）

### 実際の失敗例: xf1-v14-washi-v6開発

#### 問題
和紙でInput 240-255が全て紙白になる（階調が失われる）

#### ❌ 誤ったアプローチ（v1-v3）: プリント思考
```
「プリントで階調が失われている」
→ プリント思考: 「圧縮」されている
→ 誤った実装: ネガでも「圧縮」しよう
→ K値範囲を狭くする（28000-29400の1400 K範囲）
→ 結果: さらに悪化（階調完全消失）
```

**なぜ失敗したか**: プリントとネガの逆転関係を忘れていた

#### ✓ 正しいアプローチ（v5-v6）: ネガ思考
```
「プリントで階調が失われている」
→ ネガ思考: ネガの濃度差が小さすぎる
→ 正しい実装: ネガでは「拡張」が必要
→ K値範囲を広くする（27000-32000の5000 K範囲）
→ 結果: 階調表現が可能に（v14の4400 Kより1.14倍拡張）
```

### 実践的な思考法

**常にネガの立場で考える**:

| プリントでの目標 | ネガでの実装 |
|----------------|-----------|
| 明るくしたい | ネガを濃くする（K値を高く） |
| 暗くしたい | ネガを薄くする（K値を低く） |
| 階調を豊かにしたい | ネガの濃度差を大きくする（K値範囲を広く） |
| 階調を圧縮したい | ネガの濃度差を小さくする（K値範囲を狭く） |
| ハイライトを表現したい | ネガのハイライト域を濃くする（K値を高く） |
| シャドウを深くしたい | ネガのシャドウ域を薄くする（K値を低く） |

### AI向けの実装ガイド

#### カーブ設計時の思考プロセス

1. **ユーザーの要求を聞く**:
   - 例: 「和紙で240-255が紙白になる」

2. **ネガ思考で分析する**:
   ```
   プリント: 240-255が紙白 = 階調圧縮・消失
   ネガ: ネガの240-255の濃度差が小さすぎる
   ```

3. **ネガ側での解決策を考える**:
   ```
   プリントで階調を豊かにしたい
   → ネガの濃度差を大きくする
   → K値範囲を拡張する
   ```

4. **実装する**:
   ```python

   k_240 = 27000  # 低めに設定
   k_255 = 32000  # 高めに設定
   # 240-255の範囲 = 5000 K（拡張）
   ```

#### 禁止事項

**❌ してはいけないこと**:
- プリントの用語（「明るく」「暗く」）をそのままネガに適用する
- プリントでの「圧縮」をネガでも「圧縮」と考える
- プリントでの「拡張」をネガでも「拡張」と考える

**✓ すべきこと**:
- 常に「ネガの濃度差」と「透過光の差」で考える
- プリントでの要求を一度ネガ思考に変換してから実装する
- 「プリントで◯◯」→「ネガでは逆の△△」と明示的に変換する

### 検証方法

カーブ設計後、必ず以下を確認:

1. **K値範囲の妥当性**:
   ```python

   # 階調を圧縮したい場合: 範囲が狭いか？
   ```

2. **勾配の方向性**:
   ```python

   # → ネガでは高いK値（濃い）になっているか？
   ```

3. **ユーザーフィードバック**:
   - 設計意図を説明し、ユーザーに確認を求める

#### ルール11: 測定ベース反復改善の原則

**なぜ**: 理論計算だけでは複雑な物理化学過程（ネガ→露光→プリント）を完全に予測できない。実測データに基づく段階的改善が必要。

   ```python
   # 必須の分析項目

   - 目標濃度との偏差
   - 濃度差（ステップ）の均一性
   - 問題点の抽出
   ```

3. **問題の定量化**:

#### ルール12: K値と露光時間の相互調整

**なぜ**: K値（ネガ濃度）と露光時間は相互依存関係にあり、一方の変更は他方の調整を要求する。

**原則**:
```
K値増加（ネガ濃く）→ 露光時間短縮必要
K値減少（ネガ薄く）→ 露光時間延長必要
```

**v20-v27での実例**:
| カーブ | K[255] | 変化率 | 露光時間 | 変化率 |
|--------|--------|--------|----------|--------|
| v24 | 32820 | - | 7分30秒 | - |
| v25 | 35000 | +6.6% | 6分45秒 | -10% |
| v26 | 33500 | -4.3% | 6分45秒 | 0% |
| v27 | 36000 | +7.5% | 6分45秒 | 要検証 |

**実施方法**:
1. K_max大幅増加の場合: 露光時間を-15秒〜-30秒で試行
2. K_max減少の場合: 露光時間を+15秒〜+30秒で試行
3. 測定結果を見て微調整（±15秒単位）

#### ルール13: 紙白到達の判定基準

**なぜ**: 紙白 vs 白飛びの区別が不明確だと、真の目標到達を判定できない。

**紙白到達の基準**:
- **反射濃度 ≤ 0.09** を紙白到達と定義
- 0.11-0.15 = 白飛び（露光不足）
- 0.09以下 = 真の紙白
