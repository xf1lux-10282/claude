# P9500で9900ドライバーを使用する戦略

## アプローチの比較

### 現在の方法（手動PPD作成）
❌ **デメリット**:
- PPDを一から作成・調整が必要
- 用紙設定、マージン、給紙オプションを手動で追加
- ESC/P-Rコマンドの互換性を検証が必要
- 解像度ミスマッチなどの問題が発生

✓ **メリット**:
- P9500の12色すべてを理論的に使える可能性

### 提案：9900ドライバーを使用
✓ **メリット**:
1. **既製のPPD** - Quad9900用PPDが既に存在する可能性
2. **検証済みカーブ** - 23個のQTRカーブがすぐ使える
3. **互換性が高い** - 10色が完全一致
4. **設定不要** - 用紙、マージン、給紙オプションすべて含まれる
5. **動作保証** - 9900で実績のある設定

❌ **デメリット**:
- VI, VLMの2色が使えない（モノクロ印刷には影響なし）

## 実装方法

### 方法1: QTRインストーラーで9900を選択

QTRのインストーラーがある場合：
1. QTRインストーラーを起動
2. プリンターモデルで「Stylus Pro 9900」を選択
3. P9500を接続したまま進める
4. 自動的にQuad9900のPPDとカーブがインストールされる

### 方法2: 既存のQuad9900設定を流用

システムにQuad9900のPPDがあるか確認：

```bash
# PPDを探す
find /Library/Printers -name "*9900*.ppd*" 2>/dev/null

# または
ls /Library/Printers/QTR/ppd/ | grep -i 9900
```

見つかった場合：
1. PPDをコピーして新しいプリンターキューを作成
2. P9500の物理デバイスを指定
3. Quad9900カーブがそのまま使える

### 方法3: macOSのプリンター追加で9900を選択

システム設定 → プリンタとスキャナ：
1. 「+」でプリンター追加
2. P9500を検出
3. ドライバー選択で「Stylus Pro 9900」を手動選択
4. QTR版があればそれを選択

## 技術的根拠

### プリンターエンジンの互換性

**9900とP9500の共通点**:
- ESC/P-Rプロトコル（同一系統）
- 同じ物理ヘッド幅（44インチ）
- 同じ解像度（最大2880dpi）
- 同じ給紙機構

**違い**:
- インクセット（10色 vs 12色）
- ファームウェア（P9500の方が新しい）

**結論**: ドライバーレベルでは9900として認識させても、
ハードウェア制御（ESC/P-R）は互換性が高い。

### インク配置の互換性

9900の10色はP9500の12色中の10色と**完全一致**：

```
9900:  K  C  M  Y  LC  LM  LK  LLK  OR  GR
       ↓  ↓  ↓  ↓  ↓   ↓   ↓   ↓    ↓   ↓
P9500: K  C  M  Y  LC  LM  LK  LLK  OR  GR  VI  VLM
                                            (未使用)
```

QTRは指定された10色のみを使うため、VI/VLMが無視されても問題なし。

## 推奨手順

### ステップ1: 既存のP9550_QTRを削除

現在の手動設定プリンターを一旦削除：
```bash
lpadmin -x P9550_QTR
```

### ステップ2: 9900 PPDの確認

```bash
# 9900のPPDを探す
find /Library/Printers -name "*9900*.ppd*" -o -name "*Quad9900*.ppd*"

# QTRのppd symlink経由でも探す
ls -la /Library/Printers/QTR/ppd/ | grep 9900
```

### ステップ3: 新しいプリンターキューを作成

#### オプションA: PPDが見つかった場合
```bash
# 9900 PPDを使ってP9500用キューを作成
lpadmin -p P9500_as_9900 \
  -v "usb://EPSON/SC-P9500%20Series?serial=XXXXXXX" \
  -P /path/to/Quad9900.ppd \
  -E
```

#### オプションB: PPDが見つからない場合
システム設定のGUIから：
1. プリンタとスキャナ → 「+」
2. P9500を選択
3. 「ドライバー」で「その他を選択」
4. Stylus Pro 9900を手動で探して選択

### ステップ4: カーブの配置確認

```bash
# Quad9900カーブがあることを確認
ls /Library/Printers/QTR/quadtone/Quad9900/

# QuadP9900ディレクトリが存在するか
ls -la /Library/Printers/QTR/quadtone/ | grep 9900
```

カーブは既にQuad9900ディレクトリにあるので、
PPDがQuad9900を参照していれば自動的に使える。

### ステップ5: テスト印刷

```bash
# 利用可能なカーブを確認
lpoptions -p P9500_as_9900 -l | grep ripCurve

# テスト印刷
lpr -P P9500_as_9900 \
  -o PageSize=A4 \
  -o ripCurve1=UCmk-HanPhotoRag-Cool \
  /tmp/test_image.tif
```

## リスクと対策

### リスク1: PPDが見つからない
**対策**: QTRの最新版を再インストールし、9900を選択

### リスク2: デバイス認識の問題
**対策**: USB IDをPPDで上書き（必要に応じて）

### リスク3: ファームウェアの違い
**対策**: ESC/P-Rは下位互換性があるため、通常は問題なし

## まとめ

**最善の戦略**: 
1. Quad9900のPPDを探す
2. P9500の物理デバイスで9900ドライバーを使用
3. 既存の23個のカーブをそのまま活用

**利点**:
- 設定時間: 1時間 → 5分
- 動作確実性: 不明 → 高い
- カーブ数: 1個 → 23個

この方法で進めることを**強く推奨**します。
