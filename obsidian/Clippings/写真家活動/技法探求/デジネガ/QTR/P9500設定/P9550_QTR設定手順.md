# EPSON P9550 QTR設定手順

**作成日**: 2026年4月4日
**対象プリンター**: EPSON SureColor P9500 Series (P9550)
**目的**: P700用xf1-v28カーブをP9550でも使用可能にする

---

## 事前準備（完了済み）

### 1. システム環境
```bash
# プリンタードライバー確認
lpstat -p | grep P9500
# → EPSON_SC_P9500_Series がインストール済み
```

### 2. QTRディレクトリ構造
```
/Library/Printers/QTR/quadtone/
├── QuadP700/                    # P700用（既存）
│   └── PX1V-PtPd-xf1-v28.quad  # 現在使用中のカーブ
└── QuadP9500/                   # P9550用（新規作成済み）
    ├── PX1V-PtPd-xf1-v28-P9500.quad  # P700のv28を移植
    └── UC*.quad (23ファイル)          # UltraChrome標準カーブ
```

### 3. カーブファイル仕様
- **ベース**: P700用 xf1-v28
- **変更点**: ヘッダー情報のみ更新（プリンター名をP9500に変更）
- **カーブデータ**: P700と同一（K,C,M,Y,LC,LM,LK,LLK,V,MK 10ch全256値）

## 作業場所での設定手順

### ステップ1: プリンター接続確認

1. P9550の電源を入れる
2. USBまたはネットワーク接続を確認
3. ターミナルで確認:
```bash
lpstat -p | grep P9500
```

**期待する出力**:
```
printer EPSON_SC_P9500_Series is idle. enabled since ...
```

### ステップ2: Print-Tool起動

```bash
open -a "Print-Tool"
```

または Applications フォルダから起動

### ステップ3: Print-Tool設定

#### 基本設定
1. **Printer**: ドロップダウンから選択
   - `EPSON SC-P9500 Series` を選択

2. **Curve**: ドロップダウンから選択
   - `PX1V-PtPd-xf1-v28-P9500.quad` を選択
   - （標準カーブも選択可: UCmk-raw-neut.quad等）

3. **ICC Profile**:
   - `No Color Management` を選択
   - ※QTRカーブを直接使用するため

4. **Paper Size**:
   - 使用する用紙サイズを選択
   - P9550は最大44インチ幅対応

#### 印刷設定
- **Media Type**: 用紙タイプに合わせる（通常はプリンタードライバー設定）
- **Quality**: High Quality推奨
- **Color Mode**: Black & White（モノクロ）

### ステップ4: テスト印刷

#### 推奨テストパターン
1. **段階露光用ステップウェッジ**
   - Input値: 0, 8, 16, 24, 32, ..., 248, 255 (32段階)
   - P700での結果と濃度比較

2. **グレースケール**
   - スムーズなグラデーション確認
   - バンディングやトーンジャンプのチェック

#### 初回テスト時の注意
- 小さいサイズでテスト（A4またはハガキサイズ推奨）
- 用紙はP700で使用しているものと同じ（Pictorico OHP推奨）
- 露光時間はP700と同じ設定から開始

## 検証ポイント

### 1. 濃度特性の比較

**測定項目**:
| Input値 | P700濃度 | P9550濃度 | 差分 | 備考 |
|---------|----------|-----------|------|------|
| 0       | -        | -         | -    | 完全白 |
| 8       | -        | -         | -    | ハイライト |
| 128     | -        | -         | -    | 中間調 |
| 255     | 0.09     | -         | -    | 目標: 用紙白 |

**目標**:
- Input 255で濃度 ≤ 0.10（用紙白に近い）
- Input 0で最大濃度（v28では約1.45-1.50）
- 中間調の滑らかさ

### 2. プリンター固有の違い

**想定される差異**:
- **ヘッドサイズ**: P9550はP700より大きい（インク吐出量が異なる可能性）
- **ドット配置精度**: 大判プリンターとA3ノビプリンターの違い
- **インク濃度**: 同じUltraChrome Pro10だが、ロット差がある可能性

**対応策**:
- 濃度差が±0.05以上ある場合 → P9550専用カーブ作成を検討
- バンディングが見られる場合 → プリンター設定を調整

### 3. 露光時間の調整

P700での露光時間: **6分30秒**（xf1-v28用）

P9550で調整が必要になる可能性:
- デジタルネガの濃度がP700と異なる場合
- 同じInput値でも異なる印画紙濃度になる場合

## トラブルシューティング

### Print-ToolでQuadP9500が表示されない

**原因**: Print-Toolが再起動されていない、またはキャッシュ問題

**解決方法**:
```bash
# Print-Toolを完全終了
killall "Print-Tool"

# QTRキャッシュをクリア（必要な場合）
sudo rm -rf /Library/Printers/QTR/cache/

# Print-Toolを再起動
open -a "Print-Tool"
```

### カーブが表示されない

**確認事項**:
```bash
# QuadP9500ディレクトリの存在確認
ls -la /Library/Printers/QTR/quadtone/QuadP9500/

# xf1-v28-P9500カーブの存在確認
ls -la /Library/Printers/QTR/quadtone/QuadP9500/PX1V-PtPd-xf1-v28-P9500.quad
```

**権限問題の場合**:
```bash
sudo chmod 644 /Library/Printers/QTR/quadtone/QuadP9500/*.quad
sudo chown root:wheel /Library/Printers/QTR/quadtone/QuadP9500/*.quad
```

### RIP/PMRエラー - 用紙種類・サイズエラー

**症状**:
- Print-Toolからジョブは送信されるがプリンターで処理されない
- エラーメッセージ: "RIP/PMRエラー" または "用紙種類・サイズエラー"

**原因1: PPDにMediaType定義がない（根本原因）**

QuadToneRIPの標準PPDはMediaType定義を含まないため、P9500のような業務用プリンターでは用紙種類情報が送信されず、RIP処理が失敗します。

**解決方法**:
1. EPSON純正PPDからMediaTypeセクションを抽出
2. QuadToneRIP PPDに追加
3. プリンターキュー再作成

**実施済み（2026年4月5日）**:
- QuadP9500_v2.ppd.gz 作成・インストール完了
- MediaType=48 (Enhanced Matte Paper) 設定済み

---

**原因2: Print-Tool設定とプリンター側設定の不一致**

**解決方法**:

1. **プリンター側の設定確認**:
   - プリンター本体の操作パネルで確認
   - 例: A4、PX/MCプレミアムマット紙

2. **Print-Toolの設定を一致させる**:
   ```
   Print-Tool設定画面で:
   - Paper Size: A4（プリンター側と同じ）
   - Media Type: Enhanced Matte Paper（MediaType=48に対応）
   ```

3. **プリンタードライバー設定も確認**:
   ```bash
   # lpoptions で現在の設定を確認
   lpoptions -p QuadP9500 | grep -i "media\|page"

   # 必要に応じて変更
   lpoptions -p QuadP9500 -o MediaType=48 -o PageSize=A4
   ```

4. **MediaType値の候補**:
   - 48: Enhanced Matte Paper（現在設定中）
   - 21: Doubleweight Matte Paper
   - 118: Glossy Film 2（OHPフィルム系の場合）
   - 0: Plain Paper（汎用）

5. **設定後にテスト印刷**:
   - 小さいテストパターンから開始
   - ジョブがプリンターに送られることを確認

**重要**: Print-Tool、プリンタードライバー、プリンター本体の3箇所で用紙設定を一致させること

---

### 印刷濃度がP700と大きく異なる

**短期対応**:
1. 露光時間を調整
   - 濃い場合: 露光時間を短くする
   - 薄い場合: 露光時間を長くする

2. Photoshopでカーブ補正
   - Input値を微調整

**長期対応**:
- P9550専用カーブを作成（xf1-v29-P9500として）
- 濃度測定データを取り、P700との差分を分析

## P9550専用カーブ作成（将来的に必要な場合）

### 判断基準
以下のいずれかに該当する場合、専用カーブを検討:
- P700とP9550の濃度差が±0.10以上
- ハイライト（Input 240-255）の再現性に問題
- シャドウ（Input 0-16）のトーンジャンプ

### 作成プロセス
1. v28-P9500を使って段階露光テスト（32段階）
2. 各Input値での濃度測定
3. P700データとの差分計算
4. Pythonスクリプトで補正カーブ生成
5. v29-P9500として新規作成

**参考スクリプト**: 作業ファイル_現行版/generate_v28.py

## 技術仕様

### インク構成（共通）
- 10色 UltraChrome Pro10
- K, C, M, Y, LC, LM, LK, LLK, V, MK

### QTRチャンネルマッピング
```
Channel 1: K (Photo Black)
Channel 2: C (Cyan)
Channel 3: M (Magenta)
Channel 4: Y (Yellow)
Channel 5: LC (Light Cyan)
Channel 6: LM (Light Magenta)
Channel 7: LK (Light Black)
Channel 8: LLK (Light Light Black)
Channel 9: V (Violet)
Channel 10: MK (Matte Black) - 未使用（Photo Black使用時）
```

### カーブ構造
- 各チャンネル256値（Input 0-255）
- Output値範囲: 0-65535
- v28特性: K[255]=40000（用紙白目標）
