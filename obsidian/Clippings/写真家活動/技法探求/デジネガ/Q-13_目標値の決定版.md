# Kodak Q-13 キャリブレーション目標値【決定版】

## VueScan 公式のQ-13デフォルト値

```
White patch:  L* 95.0
Mid patch:    L* 50.0
Black patch:  L* 10.0
```

**出典**: VueScan Scanner Calibration 設定画面
**対象**: Kodak Q-13 stepwedge

---

## これが答えです

### なぜこの値なのか

| 値 | 理由 |
|---|---|
| **L*=95** (96ではない) | Q-13の最も明るいパッチ(Step A)の実測値がL*=95付近<br>VueScanがQ-13の実際の特性に合わせて設定 |
| **L*=50** | 視覚的に中間グレー<br>Q-13のStep J-K付近に相当 |
| **L*=10** | シャドウディテールを保持できる最暗部<br>Q-13のStep S-T付近に相当 |

### 他の資料で見た数値との違い

| 数値セット | 用途 | ターゲット |
|----------|------|----------|
| **95/50/10** | **Q-13専用** | **Kodak Q-13** ⭐ |
| 96/50/10 | フィルムスキャン一般 | IT8透過原稿 |
| 95/50/5 | 印刷業界 | IT8反射原稿 |
| 96/52/3 | アーカイブ | カスタム |

## 実践での使用方法

### VueScan を使う場合

#### 方法1: デフォルト値をそのまま使う（最も簡単）
```
1. VueScan → Color タブ → Scanner calibration
2. "Gray scale target" を選択
3. デフォルトで以下が設定されている:
   White: 95
   Mid: 50
   Black: 10
4. そのまま使用 ✓
```

#### 方法2: 手動で確認・調整
```
1. 上記デフォルト値を確認
2. 必要に応じて微調整可能（通常は不要）
3. Q-13の各パッチを指定
4. プロファイル作成
```

### Epson Scan を使う場合

**目標RGB値（8bit換算）:**
```
Step A (White):  RGB = 243  (L*=95 相当)
Step J (Mid):    RGB = 128  (L*=50 相当)
Step S (Black):  RGB = 25   (L*=10 相当)
```

**16bit換算:**
```
Step A (White):  RGB = 62,194  (L*=95 相当)
Step J (Mid):    RGB = 32,768  (L*=50 相当)
Step S (Black):  RGB = 6,554   (L*=10 相当)
```

## よくある質問

### Q1: L*=95 と L*=96 のどちらが正しい？
**A: Q-13 では L*=95 が正解**

- VueScan公式がQ-13用に95を設定
- Q-13の実測値がL*=95付近
- IT8ターゲット一般では96を使う場合もある
- ターゲットによって異なる

### Q2: なぜ資料によって数値が違う？
**A: ターゲットの種類が違うから**

```
Q-13専用        → 95/50/10 ⭐
IT8透過原稿      → 96/50/10
IT8反射原稿      → 95/50/5
カスタム設定     → 任意
```

### Q3: デフォルト値を変更すべき？
**A: 通常は不要**

VueScanのデフォルト値はQ-13の実測データに基づいているため、そのまま使用推奨。

ただし、以下の場合は調整を検討:
- Q-13に付属のリファレンスデータシートがある場合
- 分光測色計で実測した場合
- 特定のワークフローに最適化する場合

### Q4: ±1や±2の違いは重要？
**A: 実用上はほぼ影響なし**

```
L*=95 vs L*=96 → 目視では区別困難
L*=50 vs L*=52 → わずかに明るい中間調
L*=10 vs L*=12 → シャドウで若干の差
```

デジネガ制作では「一貫性」の方が重要。
